## 三、关系图/关系表三档说明

1. 未提交：记 0。
2. 已提交但主要停留在工具解释、概念堆叠、没有体现关系链：按部分完成扣分。
3. 已提交且明确体现关系、流程、因果、控制点与下一步：满分。

## 四、证据 -> 结论映射提醒

1. 线索不等于证据。
2. 第三方索引不等于你本人实时做出的主动探测。
3. 开放端口或服务不等于漏洞已经成立。
4. 局部请求与响应不等于更大结论自动成立。

## 五、关键字段示例

1. 浏览器解析与输出上下文基础说明：浏览器会解析而不只是显示内容；至少2种输出位置或上下文；为什么同样输入会有不同风险；至少1个安全关注点
2. Reflected XSS Low 现象记录：实验页面或目标；输入点与即时回显；执行或页面变化现象；为什么属于即时回显路径
3. Stored XSS Low 现象记录：实验页面或目标；输入被保存的证据；后续访问中的持续影响；为什么与反射型不同
4. DOM XSS 与类型差异分析：DOM XSS 的基本现象或路径；为什么更接近前端环境变化；至少1个与 Reflected/Stored 的差异；输入—输出—解析链条解释
5. Low/Medium 对照与 XSS 防护清单：Low 与 Medium 至少1项差异；说明有限过滤的局限；输出编码或上下文安全控制；至少3条防护建议

## 六、关系图/关系表正反例提示

1. 错误写法：只列工具、术语、概念，或只列结果，不写关系。
2. 合格写法：至少写出“上一层结果如何导向下一步”。
3. 较好写法：写出“现象/输入 -> 处理/机制 -> 结果 -> 控制点 -> 下一步”。
4. 与 XSS类型差异与风险链关系图/表 相关的结论必须体现“能说明什么 / 不能替代什么 / 下一步为什么是这个”。

## 七、红线规则

1. `未脱敏`：截图、账号、Cookie 或其他敏感信息未按要求脱敏，处理方式：`lt_60`。
2. `超出授权边界`：对未授权真实目标进行 XSS 观察、验证或其他测试，处理方式：`lt_60`。
3. `证据伪造或不可追溯`：现象截图、说明与实验过程明显不一致，无法支撑结论，处理方式：`lt_60`。
4. `高度雷同或抄袭代做`：与他人作业高度雷同，或存在抄袭、代做情形，处理方式：`lt_60`。

## 八、专业校验重点

1. 是否区分服务器返回内容与浏览器执行内容
2. 是否说明 XSS 的根因是输出上下文失控，而不是单纯“输入了脚本”
3. 是否区分 Reflected、Stored、DOM XSS 的路径差异
4. 是否区分有限过滤、输出编码、CSP 等控制层次
5. 是否对授权边界、风险传播范围和防护重点给出负责任表达
