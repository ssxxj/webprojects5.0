- 课程：Web应用安全与防护

- 章节主线：从公开线索到请求证据，再到授权主动探测
- profile 来源：`chapter02_web_info_collection.json`
- 建议完成时长：5-6 小时

## 二、授权边界

1. 仅限 DVWA、本机浏览器、localhost、127.0.0.1 或教师明确授权环境。
2. 不使用任何公网真实目标作为演示对象。
3. 所有截图与说明都必须可追溯、可脱敏。

## 三、能力目标

- 区分公开线索、第三方索引线索、真实请求证据和主动探测结果
- 在授权边界内完成 Google、Whois、Shodan、社工、Burp、Nmap 六类基础任务
- 说明不同工具分别处理哪一层信息、为什么会得到这类结果
- 把六层信息组织成关系链，而不是写成孤立工具清单
- 对边界、脱敏、风险和下一步选择给出负责任表达

## 六、红线规则

1. `未脱敏`：截图或文字中保留敏感信息、Session、Token等未脱敏内容，处理方式：`lt_60`。
2. `超出授权边界`：对未授权真实目标进行抓包、扫描或其他测试，处理方式：`lt_60`。
3. `证据伪造或不可追溯`：提交内容明显无法支撑结论，或截图、记录与正文严重不一致，处理方式：`lt_60`。
4. `高度雷同或抄袭代做`：与他人作业高度雷同，或存在抄袭、代做情形，处理方式：`lt_60`。

## 七、按维度扣分规则

1. 缺少一个任务：扣该任务分值。
2. 任务已提交但关键字段不全：在该任务分值内比例扣分。
3. 六层信息综合关系图/表 已提交但主要停留在工具解释、未体现关系链：按“部分完成”扣分。
4. 学生自评表、关键字段、局部格式问题：原则上只做小幅扣分，不直接封顶。
5. AI输出审核与人工复核记录缺失：在专业校验与责任意识、表达结构中扣分，不直接按红线处理。

## 八、专业校验重点

1. 是否区分线索、证据和主动探测
2. 是否明确 Shodan 属于第三方索引而非本人实时探测

## 十二、关系图/关系表正反例提示

1. 错误写法：只列工具、术语、概念，或只列结果，不写关系。
2. 合格写法：至少写出“上一层结果如何导向下一步”。
3. 较好写法：写出“现象/输入 -> 处理/机制 -> 结果 -> 控制点 -> 下一步”。
4. 与 六层信息综合关系图/表 相关的结论必须体现“能说明什么 / 不能替代什么 / 下一步为什么是这个”。
