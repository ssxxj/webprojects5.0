## 二、授权边界与红线

1. 允许：DVWA、本机浏览器、localhost、127.0.0.1，或教师明确授权的实验环境。
2. 禁止：对未授权真实目标进行扫描、抓包、上传、注入、XSS、命令执行、认证绕过或其他测试。
3. 脱敏要求：截图中的 Cookie、SessionID、Token、账号、路径、主机信息等敏感字段必须脱敏。
4. 红线说明：
   - 未脱敏：截图、参数、账号或其他敏感信息未按要求脱敏
   - 超出授权边界：对未授权真实目标进行 SQL 注入测试或验证
   - 证据伪造或不可追溯：实验记录、现象截图或说明与实际过程严重不一致
   - 高度雷同或抄袭代做：与他人作业高度雷同，或存在抄袭、代做情形
5. 雷同/抄袭说明：
   - 与同学高度雷同、抄袭、代做，按课程诚信红线处理。

## 四、你要按四步完成

### Step 1 观察
- 你需要看到什么：与本章主线直接相关的现象、页面变化、参数、响应、路径、结构或流程。
- 你需要记录什么字段：按每个任务最小字段逐项记录，不要只贴图不说明。

### Step 2 判断
- 你需要做出什么专业判断：根据本章主线，判断风险发生在哪一层、为什么成立、哪些结论还不能直接下。
- 你为什么这样判断：必须回到截图、字段、现象和关系链，而不是直接下抽象结论。

### Step 3 证据
- 你要用什么支持判断：截图、参数、响应、状态码、页面变化、结构图、比较表、流程图等。
- 你必须标出哪些关键字段：
  - 1. 数据库与查询条件基础说明：输入条件—数据库查询—返回结果；为什么页面功能与数据库有关；输入如何进入查询条件
  - 2. DVWA SQL Injection Low 现象记录：实验目标或页面；普通输入与异常输入对比；返回差异；为什么不像正常查询变化
  - 3. 机制分析与差异比较：输入—拼接—解析—执行—回显或行为差异链条；Low与Medium差异；普通SQL注入与盲注差异；根因是结构被污染而不是字符本身
  - 4. SQL注入防护清单与解释：参数化查询；输入校验；错误处理；最小权限

### Step 4 推理
- 如果条件改变，会发生什么：说明如果某个控制点加入或移除，结果为什么会变化。
- 下一步应该做什么，为什么：写出最合理的后续验证、比较或防护动作。

其他六个维度的二级评分点（简版）：

证据质量（18）
- 真实性与可追溯性（6）：是否来自授权环境、是否真实、是否可追溯。
- 关键字段可定位性（6）：截图、记录、对比中是否能看清关键字段。
- 证据与结论对应性（6）：结论是否被证据真正支持，是否存在超范围推断。

机制解释（20）
- 因果链完整性（8）：是否讲清“输入/现象 -> 处理/机制 -> 结果”。
- 根因定位准确性（8）：是否把真正根因讲对，而不是停在表面现象。
- 差异比较能力（4）：是否能解释版本差异、条件变化或证据差异。

关系链与下一步（12）
- 关系图/关系表结构质量（4）：是否体现关系、流程、因果、控制点。
- 当前结果边界判断（4）：是否写清能说明什么、不能替代什么。
- 下一步合理性（4）：是否能从当前结果推出合理下一步。

风险与防护（10）
- 风险后果判断（4）：是否能说明风险、影响面和后果。
- 防护措施分层（4）：是否区分根因控制、暴露面控制、后果缩减。
- 防护边界说明（2）：是否知道单一措施不能替代什么。

表达与结构（10）
- 结构清晰度（4）：任务顺序、收口项、材料组织是否清楚。
- 术语与表述准确性（3）：是否存在混用、模糊或绝对化表述。
- 可读性与验收友好度（3）：教师能否快速定位、快速验收。

专业校验与责任意识（10）
- 边界与授权表达（3）：是否明确授权环境、脱敏要求和边界。
- 绝对化表述控制（2）：是否避免“已经证明漏洞”等越界结论。
- 层次区分能力（3）：是否区分线索、证据、主动探测、验证结果。
- AI输出审核与人工复核质量（2）：是否能看出真实保留、删改、核实痕迹。
