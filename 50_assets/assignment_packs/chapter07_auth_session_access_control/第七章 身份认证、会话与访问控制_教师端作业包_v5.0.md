# 第七章 身份认证、会话与访问控制 教师端作业包 v5.0

## 一、基础信息

- 课程：Web应用安全与防护
- 章节：第七章 身份认证、会话与访问控制
- 章节主线：从身份确认，到持续信任、请求来源与最终放行边界
- profile 来源：`chapter07_auth_session_access_control.json`
- 建议完成时长：4-5 小时
- 提交格式：PDF
- 文件命名：`<学号><姓名>身份认证、会话与访问控制.pdf`
- 适用环境：DVWA、本机浏览器、教师授权实验环境

## 二、授权边界

1. 仅限 DVWA、本机浏览器、localhost、127.0.0.1 或教师明确授权环境。
2. 不对任何公网真实站点做认证、会话、CSRF 或访问控制验证。
3. 截图中的账号、Cookie、SessionID、Token 等敏感字段必须脱敏、可追溯。

## 三、能力目标

- 区分认证、会话、请求来源校验与访问控制的不同职责
- 在授权环境中观察 Brute Force、Weak Session IDs、CSRF 的基础现象
- 解释认证成本、会话随机性、来源校验与权限放行的关系
- 说明为什么“已登录”不能替代来源校验和访问控制
- 用“身份—状态—来源—权限”的链条解释风险与防护

## 六、红线规则

1. `未脱敏`：截图、账号、Cookie、SessionID 或其他敏感信息未按要求脱敏，处理方式：`lt_60`。
2. `超出授权边界`：对未授权真实目标进行认证、会话或来源相关测试，处理方式：`lt_60`。
3. `证据伪造或不可追溯`：现象截图、实验过程或说明明显无法支撑结论，处理方式：`lt_60`。
4. `高度雷同或抄袭代做`：与他人作业高度雷同，或存在抄袭、代做情形，处理方式：`lt_60`。

## 七、按维度扣分规则

1. 缺少一个任务：扣该任务分值。
2. 任务已提交但关键字段不全：在该任务分值内比例扣分。
3. 认证—会话—请求来源—访问控制流程图/关系图 已提交但主要停留在工具解释、未体现关系链：按“部分完成”扣分。
4. 学生自评表、关键字段、局部格式问题：原则上只做小幅扣分，不直接封顶。
5. AI输出审核与人工复核记录缺失：在专业校验与责任意识、表达结构中扣分，不直接按红线处理。

## 八、专业校验重点

1. 是否区分认证、会话、来源校验和访问控制四个环节
2. 是否说明暴力破解更接近认证成本设计问题，而不只是弱口令问题
3. 是否说明会话值的随机性与不可预测性是持续信任的基础
4. 是否说明 CSRF 的根因是来源与意图失配，而不是密码泄露
5. 是否说明“已登录”不能替代访问控制，并对授权边界给出负责任表达

## 十二、关系图/关系表正反例提示

1. 错误写法：只列工具、术语、概念，或只列结果，不写关系。
2. 合格写法：至少写出“上一层结果如何导向下一步”。
3. 较好写法：写出“现象/输入 -> 处理/机制 -> 结果 -> 控制点 -> 下一步”。
4. 与 认证—会话—请求来源—访问控制流程图/关系图 相关的结论必须体现“能说明什么 / 不能替代什么 / 下一步为什么是这个”。
