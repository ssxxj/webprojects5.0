## 一、这次作业要完成什么

1. 本次作业对应的能力目标：
   - 解释命令执行漏洞的基本形成机制与跨层风险
   - 在 DVWA 中观察 Command Injection Low 的基础现象
   - 比较 Low 与 Medium 的控制差异，并说明简单黑名单为什么不够
   - 把命令执行与 SQL 注入放在“输入进入执行结构”这一共同框架中理解
   - 用“输入—拼接/传递—系统执行—结果返回”的链条解释风险与防护
2. 完成后你应能做到什么：
   - 能按本章主线组织证据，而不是只罗列任务。
   - 能把截图、字段、现象和专业判断对应起来。
   - 能说明自己是否使用 AI，以及是否审核、删改、核实过 AI 输出。

## 二、授权边界与红线

1. 允许：DVWA、本机浏览器、localhost、127.0.0.1，或教师明确授权的实验环境。
2. 禁止：对未授权真实目标进行扫描、抓包、上传、注入、XSS、命令执行、认证绕过或其他测试。
3. 脱敏要求：截图中的 Cookie、SessionID、Token、账号、路径、主机信息等敏感字段必须脱敏。
4. 红线说明：
   - 未脱敏：截图、命令输出、账号、路径或其他敏感信息未按要求脱敏
   - 超出授权边界：对未授权真实目标进行命令执行测试或其他验证
   - 证据伪造或不可追溯：现象截图、结果说明或实验过程明显不一致
   - 高度雷同或抄袭代做：与他人作业高度雷同，或存在抄袭、代做情形
5. 雷同/抄袭说明：
   - 与同学高度雷同、抄袭、代做，按课程诚信红线处理。

## 九、常见误区提醒

1. 关系图/关系表不是工具清单，必须体现关系链。
2. 截图不是越多越好，关键字段必须能看清。
3. 线索不等于证据，端口不等于漏洞，第三方索引不等于实时探测。
4. AI 可以用，但不审核就等于没有完成本项要求。
