# assignment_packs 目录索引 v5.0

## 1. 定位

本目录用于存放 `projects5.0` 的 Step11 正式章节作业包。

这里的资产不是讲义正文，而是：

1. 教师布置作业用的执行包
2. 学生提交用的说明单
3. 教师批改用的验收表
4. 班级复盘用的反馈模板

换句话说，本目录负责的是：

`章节讲义目标 -> 作业设计 -> 提交规范 -> 评分接口 -> 教学反馈`

1. `chapter01_http_request_response/`

## 5. 对齐与来源

本目录中的章节作业包，不是直接复制 `projects4.0` 章节讲义正文，而是：

`4.0 讲义中的章节主线、章节目标、章节项目 -> 5.0 Step11 正式执行资产`

对齐依据见：

- `第4到第8章_4.0讲义到5.0作业包对照表.md`

补充说明：

- 第一章作业包当前主要作为 `课程认知脚手架样本`，其上游依据来自：
  - `chapter01_http_request_response.json`
  - `章节讲义/chapter01_http_request_response/`
- 第二章作业包当前主要作为 `5.0 模板回归检查样本`，其上游依据来自：
  - `chapter02_web_info_collection.json`
  - 第二章两班正式教学反馈
- 第三章作业包当前主要作为 `第一个漏洞机制闭环样本`，其上游依据来自：
  - `chapter03_sql_injection.json`
  - `章节讲义/chapter03_sql_injection/`

## 6. 当前边界

1. 本目录是 `作业与评估执行面` 资产，不替代未来需要单独生成的 `5.0 综合章节讲义`。
2. 若后续章节 profile 更新，应重新生成对应章节作业包，而不是手工局部改旧文件。
3. 若目录继续扩展，建议保持“每章一个目录 + 六个标准文件 + 一个 manifest”的统一结构。

## 7. 推荐使用顺序

1. 教师备课时先看：
   - `教师端作业包`
   - `任务分值与红线摘要`
2. 发给学生时使用：
   - `学生版作业说明单`
3. 批改时使用：
   - `教师验收表`
   - `班级级教学反馈模板`
4. 做迁移核查时使用：
   - `第4到第8章_4.0讲义到5.0作业包对照表.md`
5. 做模板回归核查时使用：
   - `chapter02_web_info_collection/`
