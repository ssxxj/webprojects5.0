# 第八章 综合复盘与章节项目展示 任务分值与红线摘要 v5.0

## 一、章节主线

从章节知识回顾，到统一框架比较、项目论证与课程复盘

## 三、关系图/关系表三档说明

1. 未提交：记 0。
2. 已提交但主要停留在工具解释、概念堆叠、没有体现关系链：按部分完成扣分。
3. 已提交且明确体现关系、流程、因果、控制点与下一步：满分。

## 四、证据 -> 结论映射提醒

1. 线索不等于证据。
2. 第三方索引不等于你本人实时做出的主动探测。
3. 开放端口或服务不等于漏洞已经成立。
4. 局部请求与响应不等于更大结论自动成立。

## 五、关键字段示例

1. 课程主线回顾与统一框架图：统一框架的核心节点；至少2章内容挂接到框架中；说明至少1条共性主线；体现系统化理解
2. 典型漏洞横向比较表：至少3类漏洞比较；固定比较维度；至少1个共性与1个关键差异；防护重点比较
3. 综合项目计划单：项目主题；核心判断或论点；证据来源；结构安排或分工
4. 综合项目报告或展示材料：概念、机制、证据、防护完整；围绕主论点组织内容；至少1个结构化图表或比较图；交付物清楚可复核

## 六、关系图/关系表正反例提示

1. 错误写法：只列工具、术语、概念，或只列结果，不写关系。
2. 合格写法：至少写出“上一层结果如何导向下一步”。
3. 较好写法：写出“现象/输入 -> 处理/机制 -> 结果 -> 控制点 -> 下一步”。
4. 与 统一分析框架图/项目论证结构图 相关的结论必须体现“能说明什么 / 不能替代什么 / 下一步为什么是这个”。

## 七、红线规则

1. `未脱敏`：展示材料、截图或案例中敏感信息未按要求脱敏，处理方式：`lt_60`。
2. `超出授权边界`：综合项目使用未授权真实目标或不合规素材作为展示证据，处理方式：`lt_60`。
3. `证据伪造或不可追溯`：项目材料中的截图、数据、实验记录或结论明显无法互相支撑，处理方式：`lt_60`。
4. `高度雷同或抄袭代做`：与他人作业高度雷同，或存在抄袭、代做情形，处理方式：`lt_60`。

## 八、专业校验重点

1. 是否真正使用统一框架组织前面章节，而不是简单罗列知识点
2. 是否围绕固定维度比较漏洞的共性与差异
3. 是否做到论点、证据、结构和交付物一致
4. 是否在互评或反思中呈现证据化反馈而不是空泛评价
5. 是否对引用案例、授权边界和 AI 辅助痕迹给出负责任表达
