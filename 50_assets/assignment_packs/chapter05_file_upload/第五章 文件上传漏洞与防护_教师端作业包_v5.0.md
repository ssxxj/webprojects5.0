# 第五章 文件上传漏洞与防护 教师端作业包 v5.0

## 一、基础信息

- 课程：Web应用安全与防护
- 章节：第五章 文件上传漏洞与防护
- 章节主线：从上传链路观察，到控制链完整性设计
- profile 来源：`chapter05_file_upload.json`
- 建议完成时长：3-4 小时
- 提交格式：PDF
- 文件命名：`<学号><姓名>文件上传漏洞与防护.pdf`
- 适用环境：DVWA、本机浏览器、教师授权实验环境

## 二、授权边界

1. 仅限 DVWA、本机浏览器、localhost、127.0.0.1 或教师明确授权环境。
2. 不在公网真实站点测试上传绕过、执行链或访问路径。
3. 截图中的文件名、路径、账号信息应脱敏且可追溯。

## 三、能力目标

- 解释文件上传为什么不是一个动作，而是一条处理链
- 在 DVWA 中观察 File Upload Low 的基础风险现象
- 比较 Low 与 Medium 的控制差异，并说明单点检查为什么不够
- 区分验证、存储、访问、执行各环节的控制点
- 用“上传—验证—存储—访问—执行”的链条解释风险与防护

## 六、红线规则

1. `未脱敏`：截图、文件名、路径、账号或其他敏感信息未按要求脱敏，处理方式：`lt_60`。
2. `超出授权边界`：对未授权真实目标进行上传测试或其他验证，处理方式：`lt_60`。
3. `证据伪造或不可追溯`：现象截图、上传记录或说明与实验过程明显不一致，处理方式：`lt_60`。
4. `高度雷同或抄袭代做`：与他人作业高度雷同，或存在抄袭、代做情形，处理方式：`lt_60`。

## 七、按维度扣分规则

1. 缺少一个任务：扣该任务分值。
2. 任务已提交但关键字段不全：在该任务分值内比例扣分。
3. 文件上传链路图/风险控制关系图 已提交但主要停留在工具解释、未体现关系链：按“部分完成”扣分。
4. 学生自评表、关键字段、局部格式问题：原则上只做小幅扣分，不直接封顶。
5. AI输出审核与人工复核记录缺失：在专业校验与责任意识、表达结构中扣分，不直接按红线处理。

## 八、专业校验重点

1. 是否说明上传问题本质上是链路控制问题，而不只是文件进来这一步
2. 是否区分扩展名检查、MIME 检查、内容检查、存储隔离和执行隔离
3. 是否避免把“上传成功”直接当成漏洞成立或防护失效的全部依据
4. 是否说明前端限制不能替代服务端控制链
5. 是否对授权边界、公开访问风险和执行风险给出负责任表达

## 十二、关系图/关系表正反例提示

1. 错误写法：只列工具、术语、概念，或只列结果，不写关系。
2. 合格写法：至少写出“上一层结果如何导向下一步”。
3. 较好写法：写出“现象/输入 -> 处理/机制 -> 结果 -> 控制点 -> 下一步”。
4. 与 文件上传链路图/风险控制关系图 相关的结论必须体现“能说明什么 / 不能替代什么 / 下一步为什么是这个”。
