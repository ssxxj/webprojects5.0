## 一、这次作业要完成什么

1. 本次作业对应的能力目标：
   - 解释文件上传为什么不是一个动作，而是一条处理链
   - 在 DVWA 中观察 File Upload Low 的基础风险现象
   - 比较 Low 与 Medium 的控制差异，并说明单点检查为什么不够
   - 区分验证、存储、访问、执行各环节的控制点
   - 用“上传—验证—存储—访问—执行”的链条解释风险与防护
2. 完成后你应能做到什么：
   - 能按本章主线组织证据，而不是只罗列任务。
   - 能把截图、字段、现象和专业判断对应起来。
   - 能说明自己是否使用 AI，以及是否审核、删改、核实过 AI 输出。

## 二、授权边界与红线

1. 允许：DVWA、本机浏览器、localhost、127.0.0.1，或教师明确授权的实验环境。
2. 禁止：对未授权真实目标进行扫描、抓包、上传、注入、XSS、命令执行、认证绕过或其他测试。
3. 脱敏要求：截图中的 Cookie、SessionID、Token、账号、路径、主机信息等敏感字段必须脱敏。
4. 红线说明：
   - 未脱敏：截图、文件名、路径、账号或其他敏感信息未按要求脱敏
   - 超出授权边界：对未授权真实目标进行上传测试或其他验证
   - 证据伪造或不可追溯：现象截图、上传记录或说明与实验过程明显不一致
   - 高度雷同或抄袭代做：与他人作业高度雷同，或存在抄袭、代做情形
5. 雷同/抄袭说明：
   - 与同学高度雷同、抄袭、代做，按课程诚信红线处理。

## 四、你要按四步完成

### Step 1 观察
- 你需要看到什么：与本章主线直接相关的现象、页面变化、参数、响应、路径、结构或流程。
- 你需要记录什么字段：按每个任务最小字段逐项记录，不要只贴图不说明。

### Step 2 判断

- 你为什么这样判断：必须回到截图、字段、现象和关系链，而不是直接下抽象结论。

### Step 3 证据
- 你要用什么支持判断：截图、参数、响应、状态码、页面变化、结构图、比较表、流程图等。
- 你必须标出哪些关键字段：
  - 1. 文件上传业务流程与风险链说明：上传流程的主要阶段；至少4个控制点或风险点；为什么上传不是单一步骤；至少1个业务场景
  - 2. DVWA File Upload Low 现象记录：实验页面或目标；上传结果或保存证据；访问路径或后续处理线索；至少3个明显缺失的控制点
  - 3. Low/Medium 控制差异比较：Low 与 Medium 至少2项差异；这些差异控制了哪一环；为什么单点检查不等于完整防护；至少1个残余风险
  - 4. 上传防护清单与业务方案设计：扩展名、MIME、内容检查等验证措施；重命名、隔离存储、禁执行或访问控制；至少4条防护建议；说明每条措施对应哪一环
