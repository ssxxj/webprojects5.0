## 2. 本节唯一主线

`从统一框架走到横向比较、项目论证、展示互评与课程级反思。`

## 3. 课堂目标

1. 学生能用“输入 -> 处理 -> 输出 -> 执行 -> 身份 -> 权限”的统一框架回顾前面章节。
2. 学生能用固定维度比较多类漏洞的共性与差异。
3. 学生能让项目论点、证据、结构和交付物保持一致，并在展示与互评中证据化表达。

## 4. 先修要求与环境边界

### 先修要求

1. 已完成前七章讲义与作业基础训练。
2. 已能说出至少三章的主线和核心控制点。
3. 具备最小项目汇报、比较分析和自我反思能力。

### 环境与边界

1. 综合展示所用截图、案例、现象必须来自 DVWA、本机浏览器、localhost、127.0.0.1 或教师明确授权环境。
2. 不得为展示效果补做未授权真实目标材料。
3. 展示材料中的账号、Cookie、路径、截图和互评记录必须脱敏、可追溯。

## 5. 90 分钟节奏

| 时间 | 环节 | 目标 | 教师动作 | 学生活动 | 对应证据 |

| 50-75 | Application | 收束综合项目结构、展示和互评方法 | 巡查展示材料并做证据追问。 | 完成项目结构图和展示稿修订。 | 任务4 草稿 |

## 6. OMDM 与课堂动作对照

| OMDM 节点 | 课堂动作 | 关键提问 | 预期证据 |
| --- | --- | --- | --- |
| Observation | 并列展示多章典型现象与学生项目草稿。 | 这些材料表面不同，背后的共性结构是什么？ | 学生能挂回统一框架至少两个节点。 |
| Decision | 强制判断“这是知识点罗列还是项目论证”。 | 这张图/这一页材料到底想证明什么？ | 口头选择与理由。 |
| Evidence | 标出论点、比较维度、证据和交付物对应关系。 | 哪个证据支持你的比较结论，哪个证据不够支撑？ | 框架图、比较表、展示页证据。 |
| Mechanism | 收束“统一框架 -> 比较维度 -> 论点 -> 证据 -> 交付物”链。 | 为什么材料越多不等于项目越好？ | 项目论证结构解释。 |
| Modification | 讨论去掉某页、换一个维度或换一种论证顺序时的变化。 | 如果删掉这一页，项目主论点还站得住吗？ | 条件变化解释。 |

## 7. 板书 / 图示主骨架

```text
统一框架：输入 -> 处理 -> 输出 -> 执行 -> 身份 -> 权限
项目论证：论点 -> 证据 -> 结构 -> 交付物 -> 互评 -> 反思
比较维度：进入点 / 危险环境 / 结果类型 / 控制点 / 不能替代什么
```

## 8. 关键提问设计

### 强制选择题

1. 综合复盘更接近：A 再背一遍定义；B 用统一框架重组前面章节。
2. 项目展示最重要的是：A 页数多；B 论点、证据、结构一致。
3. 互评更接近：A 主观印象；B 基于证据的反馈。

### 证据追问

1. 哪个比较维度真正支撑了你说这两类漏洞“相同”或“不同”？
2. 哪个证据支撑了你的项目主论点，而不是只是在展示材料？

### 条件变化题

1. 如果把统一框架图删掉，项目还能形成课程级收束吗？
2. 如果展示材料很多但没有主论点，为什么项目仍然会失效？

## 9. 形成性评估

### 评估动作

给学生一组前七章现象卡和一份项目草稿，要求画出统一分析框架图，做一张横向比较表，并判断当前展示材料最多支撑到“现象罗列 / 比较分析 / 项目论证”的哪一步。

### 通过标准

1. 至少能用统一框架解释两类以上漏洞的共性。
2. 能围绕固定维度做比较，而不是并列罗列知识点。
3. 能说明项目材料中的主论点、关键证据和结构关系。

### 常见失败表现

1. 把综合复盘写成“每章定义拼接”。
2. 展示材料很多，但没有明确主论点。
3. 互评和反思停留在感想层，没有证据支持。

## 10. 与 assignment pack 的对齐收口

### 本节直接服务的任务

1. 课程主线回顾与统一框架图
2. 典型漏洞横向比较表
3. 综合项目计划单
4. 综合项目报告或展示材料

### 本节必须前置的关系图/关系表要求

1. 至少写出“输入 / 处理 / 输出 / 执行 / 身份 / 权限”统一框架。
2. 至少写出一个“共性”维度和一个“差异”维度。
3. 只列章节名、漏洞名和防护名词不算达标。

### 本节应前置的 AI 输出审核提醒

1. 不能只复制总结或比较表，要写清自己核实了哪条主论点和哪组证据关系。
2. 第八章最容易误判的是：把展示材料的堆叠直接写成项目论证成立。

## 11. 高频误区与纠偏

| 高频误区 | 触发位置 | 纠偏动作 | 是否来自 memory |
| --- | --- | --- | --- |
| 任务部分完成 | 只做回顾不做比较和论证 | 追问“这份材料到底想证明什么”。 | 是 |
| 证据不清晰 | 展示与互评 | 强制补“哪条证据支持哪条结论”。 | 是 |
| 证据与结论不对应 | 综合项目论点 | 改写为“当前证据最多支撑到哪一步”。 | 是 |
| 关系链不足 | 统一框架图/结构图收口 | 强制补“统一框架”和“项目论证结构”节点。 | 是 |
| 防护建议泛化 | 综合比较表 | 强制把结论挂回具体维度和章节位置。 | 是 |

## 12. Step 9 最小观察记录

本节结束后至少记录：

1. 学生最容易把哪个“材料堆叠”误写成项目论证。
2. 哪个比较维度最容易缺失。
3. 强制选择题里最常见的错误选项是什么。

## 13. 课后衔接

### 作业布置

1. 按 assignment pack 完成四类任务与收口项。
2. 统一框架图/结构图必须体现比较维度、主论点和结论范围。
3. AI 审核记录为必填。

### 下节前置

1. 第八章是课程收束章，后续应进入项目复盘、班级反馈和课程资产回写。

### 若本节失衡，优先调整什么

1. 如果学生仍在逐章背定义，下次应压缩章节回顾、增加统一框架挂接练习。
2. 如果学生不会把论点、证据、结构对齐，下次应加强展示稿巡查和证据追问训练。
