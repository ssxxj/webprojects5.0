## 2. 本节唯一主线

`从页面现象走到请求、状态与系统链路，再把后续章节放回统一分析框架。`

## 3. 课堂目标

1. 学生能把网页理解成系统结果，而不是孤立页面。
2. 学生能说出浏览器、服务器、数据库、请求、响应、状态维持之间的最小关系。
3. 学生能用“输入—处理—输出—执行—身份”框架描述一个简单页面场景。

## 4. 先修要求与环境边界

### 先修要求

1. 无需前置安全漏洞知识。
2. 允许学生仅有最基础的网页使用经验。

### 环境与边界

1. 所有观察对象仅限 DVWA、本机浏览器、localhost、127.0.0.1 或教师明确授权环境。
2. 第一章不做任何公网对象验证、扫描或抓包演示。
3. 第一章先建立“技术动作必须服从授权边界”的课程基线。
4. 截图中的 账号、Cookie、SessionID、路径 等敏感字段必须脱敏、可追溯。

## 6. OMDM 与课堂动作对照

| OMDM 节点 | 课堂动作 | 关键提问 | 预期证据 |
| --- | --- | --- | --- |
| Observation | 展示登录页、登录后页面和状态保持现象。 | 你看到的变化只是页面变化，还是系统处理结果？ | 学生能描述现象对象。 |
| Decision | 强制判断“网页是页面还是系统结果”。 | 登录状态是浏览器自然记住你，还是系统通过状态维持识别你？ | 口头选择与理由。 |
| Evidence | 标出输入点、请求对象、状态线索。 | 哪个对象支撑你说这里发生了请求或状态维持？ | URL、参数、Cookie、页面状态。 |
| Mechanism | 收束三层架构、请求响应、Cookie/Session。 | 为什么 HTTPS 不等于业务一定安全？为什么 Cookie 和 Session 不是同一个东西？ | 机制解释。 |
| Modification | 讨论没有服务器、没有数据库、不保存 Cookie 时会怎样。 | 如果浏览器不保存 Cookie，会发生什么？ | 条件变化解释。 |

## 7. 板书 / 图示主骨架

```text
页面现象 -> 浏览器 -> 服务器 -> 数据库 -> 返回结果
请求 / 响应 / 状态维持
输入 -> 处理 -> 输出 -> 执行 -> 身份
```

## 8. 关键提问设计

### 强制选择题

1. 判断合法性的核心更接近：A 用了什么工具；B 有没有明确授权、范围和责任。
2. 登录后系统继续识别用户，更接近：A 服务器自然记住你；B 浏览器状态标识与服务器会话记录配合。
3. HTTPS 解决的更接近：A 网站一定安全；B 传输过程保护。

### 证据追问

1. 哪个现象支撑你说“这里不只是页面显示变化”？
2. 哪个对象支撑你说“系统正在维持状态”？

### 条件变化题

1. 如果没有服务器处理，这个页面变化还会不会成立？
2. 如果换一个浏览器重新访问，为什么登录状态通常不会自动继承？

## 9. 形成性评估

### 评估动作

给学生一个简单页面场景，要求画出浏览器—服务器—数据库最小关系图，标出输入、处理、输出、身份对象，并用一句话解释 Cookie 与 Session 的分工。

### 通过标准

1. 能把页面结果放回三层架构。
2. 不把 Cookie 与 Session 写成同一个东西。
3. 不把 HTTPS 写成“业务已经安全”。

### 常见失败表现

1. 只会背术语，不会回到页面现象解释。
2. 把登录状态写成“系统自然记住用户”。
3. 把关系图画成名词罗列，没有数据流方向。

## 10. 与 assignment pack 的对齐收口

### 本节直接服务的任务

1. Web三层架构与数据流图
2. HTTP请求与响应基础记录
3. Cookie与Session状态维持机制说明
4. 输入—处理—输出—执行—身份框架应用

### 本节必须前置的关系图/关系表要求

1. 至少写出输入从哪进入、处理在哪发生、结果如何返回。
2. 至少写出状态维持对象与统一分析框架之间的关系。
3. 只列名词和定义不算达标。

### 本节应前置的 AI 输出审核提醒

1. 不允许只复制术语解释，必须写“自己核实了什么”。
2. 第一章最容易误判的是：把概念定义当成机制解释，把页面现象当成系统结论。

## 11. 高频误区与纠偏

| 高频误区 | 触发位置 | 纠偏动作 | 是否来自 memory |
| --- | --- | --- | --- |
| 任务部分完成 | 只交图不写解释 | 追问“这条箭头说明了什么”。 | 是 |
| 证据不清晰 | 状态维持说明 | 追问“哪个现象支持你这么说”。 | 是 |
| 证据与结论不对应 | HTTPS / Cookie / Session 表述 | 改写为“它最多支持到哪一步”。 | 是 |
| 关系链不足 | 系统链路图收口 | 强制补“处理位置”和“状态对象”。 | 是 |
| 边界表达不足 | 开场边界判断 | 用场景反例重讲授权范围。 | 是 |
