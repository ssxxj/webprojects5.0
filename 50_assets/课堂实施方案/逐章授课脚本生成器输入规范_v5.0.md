# 逐章授课脚本生成器输入规范 v5.0

## 1. 定位

这份规范不是课堂实施方案正文，而是未来“逐章授课脚本生成器”的稳定输入接口。

它的作用是先固定：

`哪些信息必须稳定，生成器才能持续产出可执行、可复盘、可升级的课堂实施方案。`

在 `projects5.0` 中，这份规范处于：

`章节讲义 / profile / assignment pack / memory -> 输入规范 -> 授课脚本生成器 -> 课堂实施方案`

## 2. 设计原则

### Concept

稳定输入规范的目标不是增加字段，而是把课堂设计中真正可迁移、可自动化的部分抽出来。

### Mechanism

前 8 章样本回归后，可以把课堂实施方案稳定拆成两类：

1. `结构稳定项`
   任何章节都必须出现的教学控制点，例如主线、节奏、OMDM、形成性评估、作业收口、观察记录。
2. `章节变量项`
   会按章节替换的内容，例如主线文本、强制选择题、核心误区、关系图节点、任务对齐项。

如果不把这两类分开，生成器只会变成“拼接讲义段落”，而不是“按固定教学机制填充章节变量”。

### Application

因此，生成器输入必须稳定分成 6 层：

1. 系统元数据层
2. 章节主线层
3. 班级与环境层
4. 课堂动作层
5. 作业对齐层
6. 观察与复盘层

## 3. 输入分层模型

### A. 系统元数据层

必须字段：

- `schema_version`
- `generator_target`
- `course`
- `chapter_id`
- `chapter_title`
- `duration_minutes`
- `owner`
- `updated`

作用：定义生成器正在处理哪一章、输出给谁、按什么版本工作。

### B. 章节主线层

必须字段：

- `chapter_type`
  可取：`基础章 / 观察章 / 漏洞机制章 / 综合章`
- `core_spine`
  本节唯一主线，1 句，建议不超过 30 字
- `chapter_goals`
- `prerequisites`
- `boundary_rules`

作用：保证生成器始终围绕“唯一主线”组织课堂，而不是罗列知识点。

### C. 班级与环境层

必须字段：

- `class_baseline`
- `environment`
- `authorized_scope`
- `sensitive_fields_to_mask`

建议字段：

- `common_student_confusions`
- `time_pressure_notes`

作用：让同一章节在不同班级、不同环境下，仍能生成可执行而不是抽象化的课堂方案。

### D. 课堂动作层

必须字段：

- `pace_blocks`
  固定五段：
  - 开场与边界
  - Concept
  - Mechanism
  - Application
  - 形成性评估与收口
- `omdm_map`
  固定五节点：
  - Observation
  - Decision
  - Evidence
  - Mechanism
  - Modification
- `board_skeleton`
- `key_questions`
  至少包含：
  - `forced_choice`
  - `evidence_probe`
  - `conditional_change`

作用：保证生成器输出的是课堂动作，而不是教学说明。

### E. 作业对齐层

必须字段：

- `served_tasks`
- `required_relation_graph`
- `ai_review_prompt`
- `formative_assessment`
  至少包含：
  - `action`
  - `pass_criteria`
  - `common_failures`

作用：把 Step 8 和 Step 11 接起来，避免课堂和作业脱节。

### F. 观察与复盘层

必须字段：

- `high_freq_misconceptions`
- `step9_min_observations`
- `after_class_bridge`
  至少包含：
  - `assignment`
  - `next_chapter`
  - `if_unbalanced_adjustment`

作用：把课堂执行层和 SK08 复盘层接起来。

## 5. 上游映射规则

### Concept

稳定输入不是要求教师重复填写全部内容，而是要求生成器优先从现有资产中抽取，再由教师补少量变量。

### Mechanism

推荐抽取顺序：

1. 从 `教师版讲义` 抽：
   - `chapter_type`
   - `core_spine`
   - `chapter_goals`
   - 单元主线与核心误区
2. 从 `assignment pack` 抽：
   - `served_tasks`
   - `required_relation_graph`
   - `ai_review_prompt`
3. 从 `chapter profile` 抽：
   - `boundary_rules`
   - 专业校验重点中的关键约束
4. 从 `10_memory` 抽：
   - `high_freq_misconceptions`
   - 典型纠偏动作

### Application

教师真正需要补的，一般只剩：

1. `class_baseline`
2. `environment`
3. `pace_blocks` 的具体课堂动作
4. `key_questions`
5. `step9_min_observations`

## 6. 最小可用输入集

如果未来先做轻量版“授课脚本生成器”，最小输入集至少应包括：

- `chapter_title`
- `duration_minutes`
- `core_spine`
- `chapter_goals`
- `class_baseline`
- `boundary_rules`
- `served_tasks`
- `required_relation_graph`
- `high_freq_misconceptions`

缺少这些字段时，只能生成“课堂脚本草稿”，不能生成正式 `课堂实施方案_v5.0.md`。

## 7. 质量门槛

生成器只有在输入满足以下条件时，才应输出正式方案：

1. `core_spine` 只有 1 条，不是并列主题。
2. `pace_blocks` 覆盖完整 90 分钟五段结构。
3. `omdm_map` 五节点都不为空。
4. `served_tasks` 至少 1 条，且可对应 assignment pack。
5. `required_relation_graph` 不是空泛名词表。
6. `step9_min_observations` 至少 3 条。

## 8. 推荐文件形态

推荐把这份输入规范落成 YAML 或 JSON。

建议文件名：

- `chapter_lesson_script_input_v5.yaml`
- 或 `chapter_lesson_script_input_v5.json`

在 `projects5.0` 中，推荐优先使用：

- `50_assets/课堂实施方案/逐章授课脚本输入模板_v5.0.yaml`

## 9. 与 SK06 的关系

SK06 解决“怎么把稳定输入转成课堂实施方案”。  
本规范解决“哪些输入才算稳定到足以生成课堂实施方案”。

换句话说：

- `SK06` 是转换逻辑
- 本规范是转换接口

后续如果开发真正的“逐章授课脚本生成器”，应默认读取本规范，而不是直接自由读取讲义全文。
