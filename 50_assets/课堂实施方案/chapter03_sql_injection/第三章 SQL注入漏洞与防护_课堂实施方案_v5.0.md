## 2. 本节唯一主线

`从输入影响查询结构走到结果差异，再走到结构分离防护。`

## 3. 课堂目标

1. 学生能理解 SQL 注入的根因是结构被污染，而不是字符本身危险。
2. 学生能区分“正常查询结果变化”和“查询逻辑被改变”。
3. 学生能把普通 SQL 注入、Blind SQL Injection、Low/Medium 差异放回同一条因果链。

## 4. 先修要求与环境边界

### 先修要求

1. 已完成第一章请求、响应、状态维持的最小理解。
2. 已完成第二章“线索、证据、主动探测”基本区分。
3. 能接受“页面现象背后存在后端处理与数据库查询”这一系统模型。

### 环境与边界

1. 仅限 DVWA、本机浏览器、localhost、127.0.0.1 或教师明确授权环境。
2. 不使用任何公网真实目标进行 SQL 注入实验或验证。
3. 截图、参数、账号和实验记录必须脱敏、可追溯。

## 6. OMDM 与课堂动作对照

| OMDM 节点 | 课堂动作 | 关键提问 | 预期证据 |
| --- | --- | --- | --- |
| Observation | 展示普通输入与异常输入下的结果差异。 | 这里看起来只是查不同值，还是查询逻辑被改变了？ | 学生能描述差异。 |
| Decision | 强制判断“字符问题 / 结构问题”。 | SQL 注入核心更接近字符危险，还是结构被污染？ | 口头选择与理由。 |
| Evidence | 标出输入、查询条件、结果差异和行为证据。 | 哪个差异支撑你说它不像正常查询？ | 输入点、结果差异、真假提示。 |
| Mechanism | 收束“输入 -> 拼接 -> 解析 -> 执行 -> 结果差异”。 | 为什么 payload 不是根因，结构污染才是根因？ | 因果链解释。 |
| Modification | 讨论 Blind、Medium、参数化后的变化。 | 如果没有直接回显，为什么仍可能有证据？如果改成参数化，链在哪被截断？ | 条件变化解释。 |

## 7. 板书 / 图示主骨架

```text
输入 -> 查询构造 -> 数据库解释 -> 结果差异
结构污染
根因 / 暴露面 / 后果范围
```

## 8. 关键提问设计

### 强制选择题

1. SQL 注入的根因更接近：A 输入里有几个危险字符；B 输入进入了不该进入的查询结构。
2. Blind SQL Injection 和普通 SQL 注入的最大差异更接近：A 根因不同；B 证据形式不同。
3. 参数化查询主要控制的是：A 暴露面；B 根因；C 后果范围。

### 证据追问

1. 哪个结果差异支撑你说这里不像正常查询变化？
2. 哪个现象支撑你说“即使没有直接回显，系统仍然泄露了信息”？

### 条件变化题

1. 如果把同样输入放到参数化查询里，这条链会在哪一步被打断？
2. 如果只隐藏错误信息、不解决根因，风险会怎样变化？

## 9. 形成性评估

### 评估动作

给学生一组普通输入/异常输入/盲注行为差异样例，要求判断哪些属于普通查询变化、哪些更像结构被改变，写出一条最小因果链，并标出一条防护措施控制的是根因、暴露面还是后果范围。

### 通过标准

1. 至少能正确指出 1 处“结构污染”判断依据。
2. 能说出 Blind 与普通注入共享同一根因。
3. 不把过滤某些字符直接写成最核心防护。

### 常见失败表现

1. 把 payload 或特殊字符本身写成根因。
2. 把“有回显 / 无回显”写成根因差异。
3. 把错误处理、参数化查询和最小权限写成同一层措施。

## 10. 与 assignment pack 的对齐收口

### 本节直接服务的任务

1. 数据库与查询条件基础说明
2. DVWA SQL Injection Low 现象记录
3. 机制分析与差异比较
4. SQL注入防护清单与解释

### 本节必须前置的关系图/关系表要求

1. 至少写出“输入 / 查询构造 / 数据库解释 / 结果差异 / 防护控制点”。
2. 至少写出 Blind 与普通注入共享同一根因、但证据形式不同。
3. 只列漏洞名词和防护名词不算达标。

### 本节应前置的 AI 输出审核提醒

1. 不能只复制 payload 或防护清单，要写“自己核实了哪个根因判断”。
2. 第三章最容易误判的是：把字符写成根因、把无回显写成“没有信息”。

## 11. 高频误区与纠偏

| 高频误区 | 触发位置 | 纠偏动作 | 是否来自 memory |
| --- | --- | --- | --- |
| 任务部分完成 | 只交截图不写机制 | 追问“这组差异说明了哪一步被影响”。 | 是 |
| 证据不清晰 | 普通注入与盲注比较 | 强制补“哪个现象支持你”。 | 是 |
| 证据与结论不对应 | 根因与防护表述 | 改写为“支持到哪一步就写到哪一步”。 | 是 |
| 关系链不足 | 因果链/关系图收口 | 强制补“数据库解释”和“控制点”节点。 | 是 |
| 边界表达不足 | 开场边界判断 | 明确“DVWA 以外不验证”。 | 是 |
