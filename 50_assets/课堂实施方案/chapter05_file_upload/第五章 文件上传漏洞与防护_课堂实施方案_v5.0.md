## 2. 本节唯一主线

`从上传进入系统走到验证、存储、访问与执行控制，再走到链路完整性防护。`

## 3. 课堂目标

1. 学生能理解文件上传不是单个动作，而是一条“上传 -> 验证 -> 存储 -> 访问 -> 执行”处理链。
2. 学生能区分扩展名、MIME、内容检查、存储隔离、访问控制、执行控制分别在链路中的作用。
3. 学生能把 Low / Medium 差异放回控制链，而不是只背过了哪些后缀危险。

## 4. 先修要求与环境边界

### 先修要求

1. 已完成第一章请求/响应与页面最小理解。
2. 已完成第二章线索、证据、主动探测区分。
3. 已完成第四章“输入进入输出与执行”的风险链理解，能接受“是否执行”是后续处理结果。

### 环境与边界

1. 仅限 DVWA、本机浏览器、localhost、127.0.0.1 或教师明确授权环境。
2. 不在公网真实站点测试上传绕过、执行链或访问路径。
3. 截图中的文件名、路径、账号信息应脱敏且可追溯。

## 6. OMDM 与课堂动作对照

| OMDM 节点 | 课堂动作 | 关键提问 | 预期证据 |
| --- | --- | --- | --- |

## 7. 板书 / 图示主骨架

```text
上传 -> 验证 -> 存储 -> 访问 -> 执行
验证：扩展名 / MIME / 内容
存储：目录 / 重命名 / 隔离
访问：公开可达 / 私有可达
执行：是否被解析 / 是否可执行
```

## 8. 关键提问设计

### 强制选择题

1. 文件上传风险更接近：A 只要文件到服务器就等于漏洞；B 是否形成风险要看验证、存储、访问、执行链。
2. 只检查文件后缀更接近控制：A 完整防护；B 链路中的一个弱控制点。
3. 前端限制文件类型更接近控制：A 服务端验证；B 用户界面提示。

### 证据追问

1. 哪个现象支撑你说这里只是上传成功，而不是已经证明可执行？
2. 哪个字段或路径支撑你判断风险来自存储、访问还是执行控制缺失？

### 条件变化题

1. 如果文件被自动重命名并存入不可公开访问目录，风险链会怎么变化？
2. 如果服务器检查了扩展名但没有隔离目录与执行权限，为什么风险仍可能保留？

## 9. 形成性评估

### 评估动作

给学生一组文件上传 Low / Medium 现象，要求画出最小上传链，指出缺失或新增的控制点，并判断当前证据最多支撑到“上传成功 / 可访问 / 可执行”的哪一步。

### 通过标准

1. 至少能正确写出“上传 -> 验证 -> 存储 -> 访问 -> 执行”五段链路。
2. 能区分扩展名检查和完整服务端控制链不是同一层。
3. 不把“上传成功”直接写成“漏洞已成立”。

### 常见失败表现

1. 把文件上传写成“传文件 -> 成功”两步。
2. 只讨论危险后缀，不讨论存储目录、访问路径和执行权限。
3. 把前端限制直接当作服务端安全控制。

## 10. 与 assignment pack 的对齐收口

### 本节直接服务的任务

1. 文件上传业务流程与风险链说明
2. DVWA File Upload Low 现象记录
3. Low / Medium 控制差异比较
4. 上传防护清单与业务方案设计

### 本节必须前置的关系图/关系表要求

1. 至少写出“上传 / 验证 / 存储 / 访问 / 执行”五段。
2. 至少写出一个“能说明什么”和一个“不能直接替代什么”。
3. 只列工具、后缀或防护名词不算达标。

### 本节应前置的 AI 输出审核提醒

1. 不能只复制危险扩展名清单，要写清自己核实了哪一段控制缺失。
2. 第五章最容易误判的是：把上传成功直接写成完整漏洞结论。

## 11. 高频误区与纠偏

| 高频误区 | 触发位置 | 纠偏动作 | 是否来自 memory |
