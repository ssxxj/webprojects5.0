## 2. 本节唯一主线

`从公开线索走到请求证据，再走到授权主动探测。`

## 3. 课堂目标

1. 学生能区分公开线索、第三方索引线索、真实请求证据与主动探测结果。
2. 学生能在授权边界内说明 Google / Whois / Shodan / Burp / Nmap 分别回答什么问题。
3. 学生能说清“能说明什么 / 不能替代什么 / 下一步为什么是这个”。

## 4. 先修要求与环境边界

### 先修要求

1. 已完成第一章最小系统链路理解。
2. 能识别页面输入点、URL 参数、Cookie 与页面响应差异。

### 环境与边界

1. 仅限 DVWA、本机浏览器、localhost、127.0.0.1 或教师明确授权环境。
2. 不使用任何公网真实目标作为演示对象。
3. 所有截图与说明都必须可追溯、可脱敏。

## 8. 关键提问设计

### 强制选择题

1. Shodan 的结果更接近：A 实时主动探测；B 第三方历史索引；C 已证实漏洞。
2. Burp 的教学价值更接近：A 页面截图备份；B 真实交互证据；C 漏洞最终结论。

### 证据追问

1. 你为什么认为这是“请求证据”而不是“页面现象”？
2. 哪个字段支撑你说“这里存在参数变化”？

### 条件变化题

1. 如果只有 Google / Whois / Shodan，没有 Burp，你最多能说到哪一步？
2. 如果没有授权，为什么 Nmap 不应作为本章课堂动作？

## 9. 形成性评估

### 评估动作

给学生一组混合结果样例，要求判断其属于哪一层信息，写出“能说明什么 / 不能替代什么”，并说明下一步为什么应是这个。

### 通过标准

1. 至少正确区分 4 类结果中的 3 类。
2. 至少能对 1 类结果说出“不能替代什么”。
3. 不把线索直接写成漏洞结论。

### 常见失败表现

1. 把端口开放直接写成漏洞。
2. 把 Shodan 写成自己实时探测结果。
3. Burp 截图很多，但字段没标清。

## 10. 与 assignment pack 的对齐收口

### 本节直接服务的任务

1. Google 公开页面线索记录
2. Whois 域名身份信息记录
3. Shodan 服务暴露线索记录
4. Burp 请求证据观察
5. Nmap 授权主动探测记录

### 本节必须前置的关系图/关系表要求

1. 至少写出“信息层 / 方法 / 结果类型 / 能说明什么 / 不能替代什么 / 下一步”。
2. 只列工具名和结果名不算达标。

### 本节应前置的 AI 输出审核提醒

1. 不能只写“用了 AI”，要写保留了什么、删改了什么、核实了什么。
2. 本章最容易误判的是“把线索写成证据，把证据写成漏洞结论”。

## 11. 高频误区与纠偏

| 高频误区 | 触发位置 | 纠偏动作 | 是否来自 memory |
| --- | --- | --- | --- |
| 关系链不足 | 关系图收口 | 强制要求补“不能替代什么”和“下一步”。 | 是 |
| 证据不清晰 | Burp / Nmap 字段说明 | 追问“哪个字段支持你”。 | 是 |
| 证据与结论不对应 | Shodan / Burp / Nmap 结论表达 | 改写为“支持到哪一步就写到哪一步”。 | 是 |
| 任务部分完成 | 只交截图不写机制 | 现场示范最小解释模板。 | 是 |
