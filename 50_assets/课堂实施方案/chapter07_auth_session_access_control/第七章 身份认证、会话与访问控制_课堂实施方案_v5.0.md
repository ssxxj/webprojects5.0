## 2. 本节唯一主线

`从身份确认走到持续信任、请求来源与最终放行边界，建立身份信任链。`

## 3. 课堂目标

1. 学生能区分认证、会话、请求来源校验与访问控制的不同职责。
2. 学生能理解“已登录”不等于后续请求可信，也不等于最终有权放行。
3. 学生能把 Brute Force、Weak Session IDs、CSRF 放回同一条“身份 -> 状态 -> 来源 -> 权限”链中解释。

## 4. 先修要求与环境边界

### 先修要求

1. 已完成前几章“输入进入危险结构”的主线。
2. 能接受风险并不只发生在输入瞬间，也可能发生在系统持续信任和放行决策环节。
3. 具备最小请求、Cookie、状态概念。

### 环境与边界

1. 仅限 DVWA、本机浏览器、localhost、127.0.0.1 或教师明确授权环境。
2. 不对任何公网真实站点做认证、会话、CSRF 或访问控制验证。
3. 截图中的账号、Cookie、SessionID、Token 等敏感字段必须脱敏、可追溯。

## 7. 板书 / 图示主骨架

```text
身份 -> 状态 -> 来源 -> 权限
认证：你是谁
会话：你还是你吗
来源：这是不是你的真实意图
权限：你到底能做什么
```

## 8. 关键提问设计

### 强制选择题

1. Brute Force 更接近：A 口令知识点；B 认证成本设计问题。
2. Weak Session IDs 主要破坏：A 输入校验；B 持续信任。
3. CSRF 更接近：A 密码泄露；B 来源与意图失配。

### 证据追问

1. 哪个现象支撑你说这里的问题发生在会话而不是认证？
2. 哪个现象支撑你说“已登录”仍不能说明请求可信或有权放行？

### 条件变化题

1. 如果加入验证码、限速或锁定，Brute Force 的风险链会怎样变化？
2. 如果会话足够随机但没有访问控制，为什么风险仍可能存在？

## 9. 形成性评估

### 评估动作

给学生一组 Brute Force、Weak Session IDs、CSRF 现象，要求画出最小身份信任链，指出失控环节与控制点，并判断当前证据最多支撑到“认证成本问题 / 会话问题 / 来源问题 / 权限问题”的哪一步。

### 通过标准

1. 至少能正确写出“身份 -> 状态 -> 来源 -> 权限”四段链路。
2. 能区分认证、会话、来源校验和访问控制。
3. 不把“已登录”直接写成放行依据。

### 常见失败表现

1. 把 Brute Force、Weak Session IDs、CSRF 当作互不相关的漏洞名词。
2. 把 CSRF 写成密码泄露问题。
3. 把访问控制和登录状态混在一起。

## 10. 与 assignment pack 的对齐收口

### 本节直接服务的任务

1. 认证、会话与访问控制系统模型
2. Brute Force Low/Medium 观察与分析
3. Weak Session IDs 机制说明
4. CSRF 现象与来源校验分析
5. 访问控制与身份安全加固清单

### 本节必须前置的关系图/关系表要求

1. 至少写出“身份 / 状态 / 来源 / 权限”四段。
2. 至少写出一个“能说明什么”和一个“不能直接替代什么”。
3. 只列漏洞名和防护名词不算达标。

### 本节应前置的 AI 输出审核提醒

1. 不能只复制漏洞定义和防护清单，要写清自己核实了哪一段信任链失控。
2. 第七章最容易误判的是：把已登录直接写成请求可信和权限可信。

## 11. 高频误区与纠偏

| 高频误区 | 触发位置 | 纠偏动作 | 是否来自 memory |
| --- | --- | --- | --- |
| 任务部分完成 | 只交定义不交链路 | 追问“系统为什么还要继续判断”。 | 是 |

| 防护建议泛化 | 加固清单设计 | 强制把建议挂回认证、会话、来源、权限节点。 | 是 |
