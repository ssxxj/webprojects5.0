## 2. 本节唯一主线

`从应用层输入进入系统执行环境，到跨层边界控制与后果范围收束。`

## 3. 课堂目标

1. 学生能理解命令执行漏洞的关键不是命令样例，而是输入进入了系统执行结构。
2. 学生能区分应用层现象变化与系统层执行变化。
3. 学生能把 Low / Medium 差异放回“输入限制、参数分离、替代接口、最小权限”这一控制链。

## 4. 先修要求与环境边界

### 先修要求

1. 已完成第三章“数据进入查询结构”主线。
2. 已完成第五章“上传进入处理链”主线。
3. 能接受风险会从应用层放大到系统层。

### 环境与边界

1. 仅限 DVWA、本机浏览器、localhost、127.0.0.1 或教师明确授权环境。
2. 不对任何公网真实主机做命令执行观察、验证或对比。
3. 截图中的主机信息、路径、账号等敏感内容必须脱敏、可追溯。

| OMDM 节点 | 课堂动作 | 关键提问 | 预期证据 |

## 7. 板书 / 图示主骨架

```text
输入 -> 调用方式 -> 系统执行 -> 结果返回 -> 后果范围
调用方式：直接拼接 / 受限参数 / 替代接口
输入限制 -> 参数分离 -> 替代接口 -> 最小权限 -> 结果处理
```

## 8. 关键提问设计

### 强制选择题

1. 命令执行漏洞更接近：A 会很多命令才危险；B 不可信输入进入了系统执行结构。
2. 黑名单过滤几个符号更接近：A 完整防护；B 表面限制。
3. 替代接口和最小权限更接近控制：A 表示层；B 执行结构与后果范围。

### 证据追问

1. 哪个现象支撑你说这里不只是页面回显变化？
2. 哪个位置支撑你判断输入已经进入命令整体执行结构？

### 条件变化题

1. 如果应用改用安全接口而不直接拼接系统命令，风险链会怎样变化？
2. 如果系统权限被压缩，即使结构仍有问题，后果范围会怎样变化？

## 9. 形成性评估

### 评估动作

给学生一组 Command Injection Low / Medium 现象，要求画出最小风险链，指出失控点和控制点，并判断当前证据最多支撑到“页面变化 / 系统执行变化 / 后果范围扩大”的哪一步。

### 通过标准

1. 至少能正确写出“输入 -> 拼接/传递 -> 系统执行 -> 结果返回”四段链路。
2. 能说明命令执行为什么是跨层风险。
3. 不把简单黑名单直接写成结构性防护。

### 常见失败表现

1. 把命令执行写成“命令知识点”。
2. 只讨论符号和样例，不讨论应用层到系统层的边界。
3. 把执行结果直接写成完整机制结论。

## 10. 与 assignment pack 的对齐收口

### 本节直接服务的任务

1. 应用层到系统层执行链说明
2. DVWA Command Injection Low 现象记录
3. Low / Medium 控制差异与命令结构分析
4. 改进方案与防护清单

### 本节必须前置的关系图/关系表要求

1. 至少写出“输入 / 调用方式 / 系统执行 / 结果返回 / 控制点”。
2. 至少写出一个“能说明什么”和一个“不能直接替代什么”。
3. 只列命令样例、防护名词或工具名不算达标。

### 本节应前置的 AI 输出审核提醒

1. 不能只复制绕过样例或命令清单，要写清自己核实了哪个结构失控点。
2. 第六章最容易误判的是：把返回结果变化直接写成完整漏洞机制结论。

## 11. 高频误区与纠偏

| 高频误区 | 触发位置 | 纠偏动作 | 是否来自 memory |
| --- | --- | --- | --- |
| 任务部分完成 | 只交现象不交链路 | 追问“输入如何走到系统执行环境”。 | 是 |
| 证据不清晰 | Low / Medium 对比 | 强制补“哪个现象支持跨层判断”。 | 是 |
| 证据与结论不对应 | 直接写执行漏洞已完全成立 | 改写为“当前证据最多支撑到哪一步”。 | 是 |
| 关系链不足 | 风险链/关系图收口 | 强制补“调用方式”和“控制层次”节点。 | 是 |
| 防护建议泛化 | 改进方案设计 | 强制把建议挂回参数分离、替代接口、最小权限。 | 是 |

2. 哪个控制层次最容易被学生忽略。
3. 强制选择题里最常见的错误选项是什么。

1. 按 assignment pack 完成四类任务与收口项。
2. 风险链/关系图必须体现跨层边界、控制点和结论范围。
3. AI 审核记录为必填。

### 下节前置

1. 第六章结束后，第七章进入“系统为什么信任你、为什么继续信任你、为什么最终放行”的身份信任链主线。

### 若本节失衡，优先调整什么
