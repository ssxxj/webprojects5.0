## 2. 本节唯一主线

`从输入进入输出位置走到浏览器解析执行，再走到输出控制与执行限制。`

## 3. 课堂目标

1. 学生能理解 XSS 的根因是输出上下文失控，而不是“输入了脚本”。
2. 学生能区分 Reflected、Stored、DOM 三类路径差异。
3. 学生能把 Low / Medium 差异和输出编码、输入校验、CSP 放回同一条风险链。

## 4. 先修要求与环境边界

### 先修要求

1. 已完成第一章页面、请求、状态最小理解。
2. 已完成第二章线索/证据/主动探测区分。
3. 已完成第三章“结构污染”主线，能接受“浏览器解释层”是另一条机制链。

### 环境与边界

1. 仅限 DVWA、本机浏览器、localhost、127.0.0.1 或教师明确授权环境。
2. 不使用任何公网真实目标进行 XSS 观察或验证。
3. 截图、账号、Cookie 等敏感信息必须脱敏、可追溯。

## 7. 板书 / 图示主骨架

```text
输入 -> 输出位置 -> 浏览器解析 -> 是否执行 -> 风险范围
上下文失控
Reflected / Stored / DOM
输出编码 -> 输出层；输入校验 -> 输入层；CSP / 执行限制 -> 执行层
```

## 8. 关键提问设计

### 强制选择题

1. XSS 的根因更接近：A 输入中有脚本；B 不可信内容进入了浏览器会解释执行的上下文。
2. Stored XSS 与 Reflected XSS 的关键差异更接近：A payload 长度不同；B 路径结构不同。
3. CSP 更接近控制：A 输入层；B 输出层；C 执行层。

### 证据追问

1. 哪个现象支撑你说这里不只是页面显示了输入？
2. 哪个位置支撑你判断这是 Reflected、Stored 还是 DOM 路径？

### 条件变化题

1. 如果同样内容输出到纯文本位置，为什么风险可能消失？
2. 如果只做输入过滤、不做输出编码和执行限制，风险会怎样残留？

## 9. 形成性评估

### 评估动作

给学生一组 Reflected / Stored / DOM 现象样例，要求判断它属于哪一类路径，写出最小风险链，并标出一条防护措施主要控制的是输入、输出还是执行。

### 通过标准

1. 至少能正确区分 3 类路径中的 2 类。
2. 能说明 XSS 根因不在“字符危险”，而在上下文失控。
3. 不把过滤几个输入直接写成最核心防护。

### 常见失败表现

1. 把 XSS 写成“脚本标签问题”。
2. 把 Reflected、Stored、DOM 差异写成“样例不同”而不是“路径不同”。
3. 把输出编码、输入校验、CSP 写成同一层措施。

## 10. 与 assignment pack 的对齐收口

### 本节直接服务的任务

1. 浏览器解析与输出上下文基础说明
2. Reflected XSS Low 现象记录
3. Stored XSS Low 现象记录
4. DOM XSS 与类型差异分析
5. Low/Medium 对照与 XSS 防护清单

### 本节必须前置的关系图/关系表要求

1. 至少写出“输入 / 输出位置 / 浏览器解析 / 是否执行 / 控制点”。
2. 至少写出 Reflected、Stored、DOM 三类路径差异。
3. 只列类型名词和防护名词不算达标。

### 本节应前置的 AI 输出审核提醒

1. 不能只复制脚本样例或防护清单，要写“自己核实了哪个上下文判断”。
2. 第四章最容易误判的是：把输入内容本身写成根因，把现象直接写成完整机制结论。

## 11. 高频误区与纠偏

| 高频误区 | 触发位置 | 纠偏动作 | 是否来自 memory |
| --- | --- | --- | --- |
| 任务部分完成 | 只交现象截图不写机制 | 追问“浏览器在哪一步把它当成了什么”。 | 是 |
