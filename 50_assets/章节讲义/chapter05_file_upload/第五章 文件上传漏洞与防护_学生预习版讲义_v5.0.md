# 第五章 文件上传漏洞与防护 学生预习版讲义 v5.0

## 1. 这一章先抓住什么

第五章最重要的一句话是：

`文件上传不是一个动作，而是一条处理链。`

所以本章不是记几种上传绕过方式，而是先学会：

1. 文件进入系统后会经过哪些环节
2. 哪一环做错了会产生风险
3. 为什么“上传成功”不等于结论成立
4. 为什么前端限制不能替代服务端控制链

## 2. 本章主线

本章统一按这条链理解：

`上传 -> 验证 -> 存储 -> 访问 -> 执行`

只要你能把每个风险点和控制点放回这条链里，后面的比较与防护就不会乱。

## 3. 为什么上传问题是“链路问题”

上传按钮只是入口。

真正需要思考的是：

1. 系统如何判断这是什么内容
2. 它被存到了哪里
3. 谁可以访问它
4. 它会不会被当成可执行内容处理

所以风险不只在“传进来”，更在“后续如何被信任和处理”。

## 4. 本章的两个核心现象

### File Upload Low

你要重点看：

1. 文件被接收了
2. 保存到了哪里
3. 后续如何被访问
4. 哪些控制几乎没有做

### Low / Medium 对照

你要重点看：

1. Medium 比 Low 多了什么控制
2. 这些控制属于哪一环
3. 为什么单点检查仍然可能不够

## 5. 本章最容易犯的错误

### 误区1：看到上传成功，就直接写“漏洞成立”

更稳的写法应该是：

1. 说明系统接收了不可信内容
2. 说明内容被保存或可访问
3. 再解释风险为什么可能扩大

### 误区2：把防护理解成“检查扩展名就够了”

更稳的理解应该是：

1. 验证只是其中一环
2. 还要看存储、访问和执行隔离

### 误区3：只画流程，不写风险点和控制点

本章最后的关系图/表，不是流程图抄写，而是要写出：

1. 这一环的风险点
2. 这一环的控制点
3. 能说明什么
4. 不能替代什么
5. 下一步

## 6. 本章作业会要求你完成什么

你需要完成四类任务：

1. 文件上传业务流程与风险链说明
2. DVWA File Upload Low 现象记录
3. Low/Medium 控制差异比较
4. 上传防护清单与业务方案设计

还要完成三类收口项：

1. `文件上传链路图/风险控制关系图`
2. 学生自评表
3. AI输出审核与人工复核记录

## 7. 关系图/关系表应该怎么做

至少要写出这些列：

1. 链路阶段
2. 风险点
3. 能说明什么
4. 不能替代什么
5. 控制点
6. 下一步

如果只有“上传、验证、存储、访问、执行”这五个词，不算真正达标。

## 8. 提交前先检查三件事

### 证据是否清楚

截图里至少要让老师看见：

1. 上传结果
2. 保存证据或访问路径线索
3. 关键对比点

### 结论是否超范围

不要把“上传成功”直接写成“漏洞完全成立”。

更稳的写法是：

1. `说明系统在该环节存在错误信任`
2. `说明控制链在某些环节可能缺失`

### AI 输出是否审核过

如果你用了 AI，不要只写“用了”。要写清：

1. 保留了什么
2. 删改了什么
3. 核实了什么
4. 最容易误判的边界是什么

## 9. 预习时先想三个问题

1. 为什么说上传风险更像流程控制问题，而不是单点功能问题？
2. 为什么“只检查扩展名”通常不等于真正安全？
3. 如果业务方说你的方案太麻烦，你觉得哪些控制不能省？

## 10. 预习后你至少应能说出

1. 上传链的五个基本阶段
2. File Upload Low 暴露的是哪类错误信任
3. Low / Medium 比较为什么看的是控制层次
4. 验证、存储、访问、执行分别大致控制什么
5. 为什么本章最后一定要交“风险控制关系图”
