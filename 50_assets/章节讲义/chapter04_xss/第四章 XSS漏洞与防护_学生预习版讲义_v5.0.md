# 第四章 XSS漏洞与防护 学生预习版讲义 v5.0

## 1. 这一章先抓住什么

第四章最重要的一句话是：

`XSS 的关键不是“输入了什么”，而是“浏览器把输出当成了什么”。`

所以本章不是记几个脚本样例，而是先学会：

1. 浏览器为什么不只是显示文本
2. 输出位置为什么决定风险
3. 为什么 Reflected / Stored / DOM 三类路径不同
4. 为什么防护重点在输出与执行控制

## 2. 本章主线

本章统一按这条链理解：

`输入 -> 输出位置 -> 浏览器解析 -> 是否执行 -> 风险范围 -> 控制点`

只要你能把每种 XSS 都放回这条链里，后面的比较和防护就不会乱。

## 3. 三类 XSS 先怎么分

### Reflected XSS

更像：

`输入提交后立即回到页面，再被浏览器解释`

### Stored XSS

更像：

`输入先被保存，后续再被输出和解释`

### DOM XSS

更像：

`前端脚本读取不可信输入，再把它写回 DOM`

请先记住：

这三类不是“完全不同的漏洞”，而是“路径不同，但都和危险输出/执行上下文有关”。

## 4. 为什么浏览器解析很重要

如果浏览器只是显示字符，就不会有这一章。

问题就在于浏览器会：

1. 解析 HTML
2. 解析属性
3. 解析脚本
4. 根据前端脚本继续更新 DOM

所以同样一段输入：

1. 放在纯文本位置，可能只是文本
2. 放在危险上下文，可能就变成结构或执行内容

## 5. 本章最容易犯的错误

### 误区1：把 XSS 理解成“输入里有脚本”

更稳的理解应该是：

`不可信内容进入了浏览器会解释执行的上下文`

### 误区2：把现象直接当根因

比如看到页面变化，就直接写“漏洞成立”。

你应该先说明：

1. 输入在哪里
2. 输出在哪里
3. 浏览器为什么会这样解释

### 误区3：把防护理解成“多拦几个输入”

更稳的方向是：

1. 控制输出上下文
2. 做上下文相关编码
3. 再配合输入校验和执行限制

## 6. 本章作业会要求你完成什么

你需要完成五类任务：

1. 浏览器解析与输出上下文基础说明
2. Reflected XSS Low 现象记录
3. Stored XSS Low 现象记录
4. DOM XSS 与类型差异分析
5. Low/Medium 对照与 XSS 防护清单

还要完成三类收口项：

1. `XSS类型差异与风险链关系图/表`
2. 学生自评表
3. AI输出审核与人工复核记录

## 7. 关系图/关系表不是名词对照表

你最后交的图/表，不应只写：

1. 类型名称
2. 现象名称
3. 防护名词

至少还要写出：

1. 形成路径
2. 风险点
3. 能说明什么
4. 不能替代什么
5. 防护重点
6. 下一步

如果只有“类型 + 现象 + 工具/方法”，这不算真正的关系表达。

## 8. 提交前先检查三件事

### 证据是否清楚

截图里至少要让老师看见：

1. 输入点
2. 输出位置
3. 页面变化或执行现象
4. 关键比较点

### 结论是否超范围

不要把“看到现象”直接写成“已经完全证明根因”。

更稳的写法是：

1. `说明浏览器在该位置解释了输出`
2. `说明该路径存在危险上下文控制问题`

### AI 输出是否审核过

如果你使用了 AI，不要只写“用了”。要写清：

1. 保留了什么
2. 删改了什么
3. 核实了什么
4. 最容易误判的边界是什么

## 9. 预习时先想三个问题

1. 为什么 XSS 的核心不是“有人输入了脚本”，而是“浏览器解释了输出”？
2. Reflected、Stored、DOM 三类真正的差异在“样例形式”还是“路径结构”？
3. 为什么防护重点通常不只是输入过滤，而是输出控制和执行限制？

## 10. 预习后你至少应能说出

1. 浏览器为什么不是纯文本播放器
2. 输出上下文为什么会决定风险
3. Reflected / Stored / DOM 三类路径差异
4. 输出编码、输入校验、CSP 分别更偏哪一层控制
5. 为什么本章最后一定要交“风险链关系图/表”
