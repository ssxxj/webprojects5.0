# 第六章 命令执行漏洞与防护 学生预习版讲义 v5.0

## 1. 这一章先抓住什么

第六章最重要的一句话是：

`命令执行问题的关键，不是“用了什么命令”，而是“输入进入了系统执行结构”。`

所以本章不是记几条命令样例，而是先学会：

1. 输入怎样从应用层跨到系统层
2. 为什么这是跨层风险
3. 为什么黑名单通常不等于真正防护
4. 为什么参数分离、替代接口和最小权限更关键

## 2. 本章主线

本章统一按这条链理解：

`输入 -> 拼接/传递 -> 系统执行 -> 结果返回`

只要你能把每个风险点和控制点放回这条链里，后面的比较与防护就不会乱。

## 3. 为什么这是“跨层风险”

输入原本只是页面里的一个值。

但如果系统把它带进后台调用链，它就不再只是页面参数，而会影响系统层行为。

所以本章的关键不是“页面回显”，而是：

1. 输入进入了哪条调用链
2. 输入会不会影响命令整体结构
3. 系统层后果会不会被放大

## 4. 本章的两个核心现象

### Command Injection Low

你要重点看：

1. 输入发生了变化
2. 返回结果也发生了变化
3. 这种变化为什么不只是页面显示不同

### Low / Medium 对照

你要重点看：

1. Medium 相比 Low 多了什么限制
2. 这些限制控制的是表面字符，还是执行结构
3. 为什么黑名单仍然可能不够

## 5. 本章最容易犯的错误

### 误区1：把结果差异直接当成结论

更稳的写法应该是：

1. 说明输入影响了系统执行结果
2. 再解释为什么这说明输入进入了执行结构

### 误区2：把黑名单当作完整防护

更稳的理解应该是：

1. 黑名单只是表面限制
2. 更关键的是不让输入进入命令结构
3. 还要看最小权限和后果范围控制

### 误区3：只画应用流程，不写跨层风险

本章最后的关系图/表，不只是“页面 -> 后台 -> 返回”的流程图，还要写：

1. 风险点
2. 能说明什么
3. 不能替代什么
4. 控制点
5. 下一步

## 6. 本章作业会要求你完成什么

你需要完成四类任务：

1. 应用层到系统层执行链说明
2. DVWA Command Injection Low 现象记录
3. Low/Medium 控制差异与命令结构分析
4. 改进方案与防护清单

还要完成三类收口项：

1. `应用层到系统层风险链路图/防护关系图`
2. 学生自评表
3. AI输出审核与人工复核记录

## 7. 关系图/关系表应该怎么做

至少要写出这些列：

1. 链路阶段
2. 风险点
3. 能说明什么
4. 不能替代什么
5. 控制点
6. 下一步

如果只有“输入、执行、输出”三个词，不算真正达标。

## 8. 提交前先检查三件事

### 证据是否清楚

截图或记录里至少要让老师看见：

1. 输入点
2. 结果差异
3. 系统执行迹象
4. 关键比较点

### 结论是否超范围

不要把“结果变了”直接写成“大结论”。

更稳的写法是：

1. `说明输入影响了系统执行行为`
2. `说明输入可能进入了执行结构`

### AI 输出是否审核过

如果你用了 AI，不要只写“用了”。要写清：

1. 保留了什么
2. 删改了什么
3. 核实了什么
4. 最容易误判的边界是什么

## 9. 预习时先想三个问题

1. 为什么命令执行比普通页面展示问题更危险？
2. 为什么“禁止几个符号”通常不如“避免让输入进入命令结构”更可靠？
3. 如果业务必须调用系统命令，你觉得哪类控制最不能省？

## 10. 预习后你至少应能说出

1. 应用层和系统层的大致区别
2. 命令执行为什么是跨层风险
3. Low / Medium 比较为什么看的是控制层次
4. 参数分离、替代接口、最小权限大致分别控制什么
5. 为什么本章最后一定要交“风险链路图/防护关系图”
