# 第七章 身份认证、会话与访问控制 学生预习版讲义 v5.0

## 1. 这一章先抓住什么

第七章最重要的一句话是：

`用户已经登录，不等于后续所有请求都可信、所有动作都应放行。`

所以本章不是把几个问题背下来，而是先学会：

1. 认证在确认什么
2. 会话在维持什么
3. 来源校验在防什么
4. 访问控制最终决定什么

## 2. 本章主线

本章统一按这条链理解：

`身份 -> 状态 -> 来源 -> 权限`

只要你能把每个风险点和控制点放回这条链里，后面的比较与防护就不会乱。

## 3. 为什么这一章是“身份信任链”

系统从登录到敏感操作，至少要连续解决四件事：

1. 这个人是谁
2. 系统怎么继续信任还是他
3. 这个请求是不是他的真实意图
4. 他到底有没有权限做这个动作

所以本章不是独立的几个漏洞，而是同一条信任链上的不同薄弱点。

## 4. 本章的三个核心现象

### Brute Force

你要重点看：

1. 系统是否允许低成本重复尝试
2. 这为什么不只是“用户密码弱”

### Weak Session IDs

你要重点看：

1. 会话值在哪里出现
2. 为什么可预测会破坏持续信任

### CSRF

你要重点看：

1. 用户明明没主动发起这个请求
2. 为什么系统仍然信任了它

## 5. 本章最容易犯的错误

### 误区1：把“已登录”当作所有后续信任依据

更稳的理解应该是：

1. 登录只解决身份确认
2. 后面还要看会话、来源和权限

### 误区2：把 Brute Force 只看成弱口令问题

更稳的理解应该是：

1. 口令强度重要
2. 但系统让错误尝试成本太低，同样会放大风险

### 误区3：把 CSRF 当作密码泄露

更稳的理解应该是：

1. 这不是密码问题
2. 而是系统错误信任了带着有效会话的请求

## 6. 本章作业会要求你完成什么

你需要完成五类任务：

1. 认证、会话与访问控制系统模型
2. Brute Force Low/Medium 观察与分析
3. Weak Session IDs 机制说明
4. CSRF 现象与来源校验分析
5. 访问控制与身份安全加固清单

还要完成三类收口项：

1. `认证—会话—请求来源—访问控制流程图/关系图`
2. 学生自评表
3. AI输出审核与人工复核记录

## 7. 关系图/关系表应该怎么做

至少要写出这些列：

1. 流程阶段
2. 风险点
3. 能说明什么
4. 不能替代什么
5. 控制点
6. 下一步

如果只有“登录、会话、CSRF、权限”这些名词，不算真正达标。

## 8. 提交前先检查三件事

### 证据是否清楚

截图或记录里至少要让老师看见：

1. 实验现象
2. 关键流程节点
3. 关键比较点

### 结论是否超范围

不要把现象直接写成大结论。

更稳的写法是：

1. `说明认证成本设计过低`
2. `说明持续信任基础变弱`
3. `说明请求来源未被额外校验`
4. `说明放行边界设计不足`

### AI 输出是否审核过

如果你用了 AI，不要只写“用了”。要写清：

1. 保留了什么
2. 删改了什么
3. 核实了什么
4. 最容易误判的边界是什么

## 9. 预习时先想三个问题

1. 为什么“已登录”不能替代后续所有安全判断？
2. 为什么 Brute Force、Weak Session IDs、CSRF 看起来不同，但其实都在削弱同一条信任链？
3. 如果系统做了登录，也做了会话，但没有访问控制，还会出什么问题？

## 10. 预习后你至少应能说出

1. 认证、会话、来源校验、访问控制各自大致负责什么
2. Brute Force 为什么更接近认证成本问题
3. Weak Session IDs 为什么会破坏持续信任
4. CSRF 为什么不是密码泄露问题
5. 为什么本章最后一定要交“身份信任链流程图/关系图”
