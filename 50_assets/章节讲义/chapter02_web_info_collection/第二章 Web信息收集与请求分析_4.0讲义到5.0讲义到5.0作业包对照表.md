## 4. 5.0 新增升级点

与 4.0 相比，第二章 5.0 讲义新增了这些系统约束：

1. `关系图/关系表` 不再作为形式性收口，而被提升为结构化理解证据。
2. `证据 -> 结论映射` 被前置到讲义与作业包，而不是只在批改后发现问题。
3. `AI输出审核与人工复核记录` 被纳入讲义要求，用于训练责任意识。
4. `R2 / E1 / E2 / T2` 高频误区已经回写到讲义层，而不是停留在反馈报告里。

## 5. 迁移结论

第二章 5.0 讲义不是 4.0 讲义的简单改写，而是：

`保留 4.0 的章节主线与 OMDM 骨架 -> 接入 5.0 assignment pack 的执行要求 -> 再接入 memory 的高频误区回流`

因此，它可以作为 `projects5.0` 正式讲义迁移的第一章回归样本。
