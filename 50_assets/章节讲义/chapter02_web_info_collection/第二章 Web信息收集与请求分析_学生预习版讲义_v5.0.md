# 第二章 Web信息收集与请求分析 学生预习版讲义 v5.0

## 1. 本章先学什么

第二章不是让你“会更多工具”，而是让你先学会：

1. 面对一个页面或系统时，先观察什么
2. 哪些结果只能算线索，哪些更接近证据
3. 哪些动作必须建立在授权前提下

本章主线只有一句话：

`从公开线索到请求证据，再到授权主动探测`

## 2. 为什么这一章重要

后面的漏洞分析不是盲试。

如果你不会：

1. 定位页面中的输入点、输出点和状态对象
2. 看请求与响应里的关键字段
3. 区分线索、证据和主动探测

那么后面章节就很容易变成“只会操作，不会解释”。

## 3. 本章要分清的四类结果

### Concept

本章最重要的不是记工具名，而是分清结果层级。

### Mechanism

如果层级混了，就会把端口写成漏洞、把线索写成证据、把第三方索引写成自己实时探测结果。

### Application

请先记住这四类：

1. `公开线索`
   例如 Google 搜索结果
2. `身份/背景线索`
   例如 Whois 字段
3. `第三方索引线索`
   例如 Shodan 的历史索引结果
4. `真实请求证据与主动探测结果`
   例如 Burp 请求、Nmap 在授权环境中的扫描结果

## 4. 本章最小观察框架

当你打开一个页面时，先问自己五个问题：

1. 页面上有哪些输入点
2. 页面上有哪些输出区域
3. 页面动作会产生什么请求
4. 请求和响应里有哪些关键字段
5. 这些结果属于线索、证据还是主动探测

## 5. 本章作业会要求你完成什么

你需要完成六类任务：

1. Google 公开页面线索记录
2. Whois 域名身份信息记录
3. Shodan 服务暴露线索记录
4. 社交工程风险场景分析
5. Burp 请求证据观察
6. Nmap 授权主动探测记录

还要完成三类收口项：

1. 六层信息综合关系图/表
2. 学生自评表
3. AI输出审核与人工复核记录

## 6. 提交前最容易出错的地方

### 误区1：图已经交了，但其实没有关系

只列工具名、概念名、结果名，不写：

1. 能说明什么
2. 不能替代什么
3. 下一步为什么是这个

这种情况不算关系表达达标。

### 误区2：截图很多，但证据点不清楚

你需要让教师一眼看到：

1. Method
2. URL/路径
3. 参数位置
4. Cookie/Header
5. 状态码或返回含义

### 误区3：结论超范围

以下写法要避免：

1. `Shodan 证明了目标当前开放这个端口`
2. `开放端口说明已经存在漏洞`
3. `页面有变化，所以漏洞已成立`

更稳的写法是：

1. `这是第三方索引线索`
2. `这说明目标可能暴露了某个服务`
3. `这说明输入变化引起了系统处理差异`

## 7. 关系图/关系表应该怎么做

至少写出这些列：

1. 信息层
2. 方法/对象
3. 结果类型
4. 能说明什么
5. 不能替代什么
6. 下一步

最低合格标准不是“画了表”，而是“写出了从上一层走到下一层的理由”。

## 8. AI输出审核与人工复核记录怎么写

不要只写“用了 AI”或“没用 AI”。

要写清：

1. 保留了什么内容，为什么保留
2. 删改了什么内容，为什么删改
3. 自己核实了哪几个专业判断
4. 哪个边界问题最容易误判

本项的目标不是调查你是否用了 AI，而是检查你是否会审核工具输出。

## 9. 预习时先思考三个问题

1. 为什么 Google、Whois、Shodan、Burp、Nmap 不能被当作同一层工具来看？
2. 为什么 Burp 更接近技术证据，但它的结论仍然不能无限放大？
3. 为什么第二章最后一定要画关系图，而不是把六个工具再列一遍？

## 10. 预习后你至少应能说出

1. 什么是线索
2. 什么是证据
3. 什么是主动探测
4. 为什么第二章所有实验都必须有授权边界
5. 为什么本章真正训练的是“观察与判断”，不是“堆工具”
