# 第三章 SQL注入漏洞与防护 学生预习版讲义 v5.0

## 1. 这一章先抓住什么

第三章最重要的一句话是：

`SQL 注入的关键不是“输入了什么字符”，而是“输入影响了查询结构”。`

所以本章不是记 payload，而是先学会：

1. 页面输入为什么会和数据库查询有关
2. 为什么查询结果变化有时不像正常业务变化
3. 为什么真正的防护是结构分离，而不是多拦几个字符

## 2. 本章主线

本章统一按这条链理解：

`输入 -> 拼接 -> 解析 -> 执行 -> 回显/行为差异 -> 防护`

只要你能把普通 SQL 注入、盲注、Low/Medium 差异都放回这条链里，本章就不会学乱。

## 3. 本章最小模型

先记住最小关系：

1. 用户输入一个条件
2. 系统把这个条件带入查询
3. 数据库按查询结构解释并执行
4. 页面返回结果或行为差异

关键不是“结果变了”，而是：

`结果为什么变得不像正常查询变化`

## 4. Low、Medium、Blind 先怎么分

### Low

重点是让你看见：

`输入变化 -> 查询结果异常变化`

### Medium

重点是让你看见：

`做了一些限制，不等于根因被切断`

### Blind

重点是让你看见：

`即使没有直接回显，行为差异也可能是证据`

请先记住：

这三者不是三套完全独立的问题，而是：

`同一根因在不同证据条件下的表现`

## 5. 本章最容易犯的错误

### 误区1：把 payload 或特殊字符当成根因

更稳的理解应该是：

`根因是输入进入了不该进入的查询结构`

### 误区2：把结果变化直接写成“漏洞成立”

你应该先说明：

1. 输入是什么
2. 正常结果是什么
3. 异常结果哪里不同
4. 为什么这种不同更像“查询逻辑被改变”

### 误区3：把防护理解成“过滤更多字符”

更稳的方向是：

1. 参数化查询
2. 输入校验
3. 错误处理
4. 最小权限

而且要说明这些措施分别控制哪一层。

## 6. 本章作业会要求你完成什么

你需要完成四类任务：

1. 数据库与查询条件基础说明
2. DVWA SQL Injection Low 现象记录
3. 机制分析与差异比较
4. SQL注入防护清单与解释

还要完成三类收口项：

1. `SQL注入因果链/防护关系图`
2. 学生自评表
3. AI输出审核与人工复核记录

## 7. 因果链/关系图不是术语表

你最后交的图，不应只写：

1. SQL 注入
2. 盲注
3. 参数化查询
4. 错误处理

至少还要写出：

1. 输入如何进入查询
2. 查询如何被解释执行
3. 结果差异怎么形成
4. 防护卡在链路的哪一段
5. 下一步为什么是这个

如果只有“概念 + 名词”，这不算真正的关系表达。

## 8. 提交前先检查三件事

### 证据是否清楚

截图或记录里至少要让老师看见：

1. 输入条件
2. 正常与异常对比
3. 返回差异
4. 你据此做出的判断

### 结论是否超范围

不要把：

`有差异 -> 已经完整证明所有机制`

写成结论。

更稳的写法是：

1. `这说明查询结果出现了异常差异`
2. `这更像输入影响了查询逻辑`
3. `这支持结构污染的判断`

### AI 输出是否审核过

如果你使用了 AI，不要只写“用了”。

要写清：

1. 保留了什么
2. 删改了什么
3. 核实了什么
4. 哪个边界最容易误判

## 9. 预习时先想三个问题

1. 为什么 SQL 注入的关键不是“字符危险”，而是“结构被污染”？
2. 为什么盲注和普通注入共享同一根因，只是证据形式不同？
3. 为什么真正有效的防护必须回到参数化、错误处理和最小权限这些控制链？

## 10. 预习后你至少应能说出

1. 页面输入为什么会和数据库查询有关
2. 为什么“结果变化”不一定等于“正常查询变化”
3. 为什么 SQL 注入根因是结构问题而不是字符问题
4. 盲注与普通注入最大的差异在证据层而不是根因层
5. 为什么本章最后一定要交 `因果链/防护关系图`
