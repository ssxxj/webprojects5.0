# 课程项目化作业评估母规则 v5.0

## 1. 定位

### Concept
本规则是《Web应用安全与防护》所有章节作业的课程级母规则。

它不是某一章的评分细则，而是整个课程的统一评估底座。

### Mechanism
所有章节作业都必须同时服务三件事：

1. `认证`
   判断学生是否达成本章能力目标。
2. `诊断`
   判断学生卡在任务执行、证据表达、机制理解、边界意识、关系链能力还是专业判断。
3. `反馈`
   将个体作业结果转化为班级级教学反馈，反向修正作业设计、课堂讲解和章节模板。

### Application
后续每章都必须以“母规则 + 章节变量”的方式运行，而不是重新发明一套评分口径。

## 2. 五类能力目标

每章作业至少要映射以下五类能力目标：

1. 任务执行能力
2. 证据表达能力
3. 机制理解能力
4. 边界与责任能力
5. 关系链与下一步能力

## 3. 双层评估框架

### 第一层：成绩层
用于给出学生分数。

默认维度：
1. 任务完成度
2. 证据质量
3. 机制解释
4. 关系链与下一步
5. 风险与防护
6. 表达与结构
7. 专业校验与责任意识

默认总分按 `100 分制` 计算。

其中：

1. `任务完成度 20 分`
   由章节任务分值拆分表直接组成，回答“有没有完成、完成到什么程度”。
2. 其余 `80 分`
   由证据质量、机制解释、关系链与下一步、风险与防护、表达与结构、专业校验与责任意识共同组成，回答“是否做对、是否做得好、是否做得负责任”。

禁止把“任务分值拆分表合计 20 分”误写成“本章总分 20 分”。

学生端、教师端和自动生成产物必须显式写明：

`20 分 = 任务完成度子分`
`100 分 = 课程统一总评`
`红线按 100 分总评执行`

### 第二层：诊断层
用于生成教学反馈。

默认输出：
1. 错误标签
2. 高风险行为标签
3. 正反样本
4. 班级高频问题统计
5. 下一轮教学建议

## 4. 红线与维度扣分分离

### 红线问题
以下问题允许直接压到 60 分以下：
1. 敏感信息未脱敏
2. 超出授权边界
3. 伪造、抄袭、代做
4. 与他人作业高度雷同
5. 课程明确禁止的违规实验

### 非红线问题
以下问题一律按维度扣分，不得直接封顶：
1. 某个任务缺失或只完成部分
2. 关键字段不全
3. 综合关系图/表关系表达不足
4. 自评表缺失
5. 机制解释较弱

## 5. 七维默认二级评分点

### 5.1 任务完成度（20）

由章节任务分值表直接组成。

默认包含：

1. 任务 1
2. 任务 2
3. 任务 3
4. 任务 4（如有）
5. 关系图 / 关系表
6. 学生自评表

它主要回答：

1. 是否提交
2. 是否完成
3. 是否部分完成
4. 缺了哪一项

它不单独承担“机制是否讲对、证据是否足够、边界是否稳”的职责。

### 5.2 证据质量（18）

1. 真实性与可追溯性（6）
2. 关键字段可定位性（6）
3. 证据与结论对应性（6）

### 5.3 机制解释（20）

1. 因果链完整性（8）
2. 根因定位准确性（8）
3. 差异比较能力（4）

### 5.4 关系链与下一步（12）

1. 关系图 / 关系表结构质量（4）
2. 当前结果边界判断（4）
3. 下一步合理性（4）

### 5.5 风险与防护（10）

1. 风险后果判断（4）
2. 防护措施分层（4）
3. 防护边界说明（2）

### 5.6 表达与结构（10）

1. 结构清晰度（4）
2. 术语与表述准确性（3）
3. 可读性与验收友好度（3）

### 5.7 专业校验与责任意识（10）

1. 边界与授权表达（3）
2. 绝对化表述控制（2）
3. 线索 / 证据 / 主动探测层次区分能力（3）
4. AI 输出审核与人工复核质量（2）

## 6. 综合关系图/表的统一口径

所有章节中，只要存在“关系图/关系表/流程图/汇总图”类收口项，统一按三档处理：

1. 未提交：0 分
2. 已提交但只停留在工具罗列或概念抄写：给部分分
3. 已提交且体现对象、层次、边界、结果类型、不能替代什么、下一步：给足分

“综合关系图”和“综合关系表”都属于有效形式。

## 7. AI 输出审核与人工复核记录

所有作业默认必须包含：
`AI输出审核与人工复核记录（必填）`

默认字段：
1. 本作业是否使用 AI：未使用 / 局部使用 / 深度使用
2. 若使用 AI，AI 主要帮助了哪些环节
3. 我保留了 AI 输出中的哪 2 处内容，为什么保留
4. 我删改了 AI 输出中的哪 2 处内容，为什么删改
5. 我自行核实过的 3 个专业判断
6. 我认为最容易误判的一处边界问题

若未使用 AI，应改填：
1. 本作业未使用 AI
2. 我主动删掉或放弃的 2 处思路
3. 我自行核实过的 3 个专业判断
4. 我认为最容易误判的一处边界问题

## 8. 输出标准

每次批改后至少产生三类输出：

1. 学生端：
   等级、优点、需要完善的地方、如何完善
2. 教师端：
   分项得分、错误标签、高风险标记
3. 班级端：
   班级平均分、各维度平均分、高频问题、下轮教学建议

## 9. 接入原则

本母规则接入后，所有章节作业都应遵循：

1. 先写章节目标对齐表
2. 再写任务与分值拆分
3. 再写红线规则
4. 再写质量维度
5. 再写诊断标签
6. 最后生成教师包与学生包
