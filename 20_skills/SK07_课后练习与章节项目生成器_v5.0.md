# SK07 课后练习与章节项目生成器 v5.0

## Purpose

把单元目标和章节目标稳定转成：

1. 教师端作业包
2. 学生版作业说明单
3. 任务分值拆分表
4. 红线与维度扣分表
5. AI 输出审核与人工复核记录
6. 诊断标签
7. 班级级反馈模板

与 4.0 相比，5.0 的重点不是“作业能发出去”，而是“作业能反哺教学”。

## Input Contract

必须输入：
1. 本节或本章能力目标
2. 章节主线
3. 授权边界与脱敏要求
4. 本章核心任务
5. 本章关系图/关系表要求
6. 本章专业校验重点
7. 本章红线规则

若 `10_memory` 已有相关章节或相近任务的沉淀，生成前还应优先查看：

1. `10_memory/诊断标签索引_v5.0.md`
2. `10_memory/误区库_v5.0.md`
3. `10_memory/案例库_v5.0.md`
4. `10_memory/班级教学反馈索引_v5.0.md`

目的不是复制旧作业，而是先吸收：

1. 上一轮学生最常见的误区
2. 哪些字段最容易漏
3. 哪些正例结构值得保留
4. 哪些模板位置已经被证明需要加强

## Process

1. 先检查 `10_memory` 中是否已有本章或相近章节的误区、标签和案例沉淀。
2. 将目标转成可观察、可提交、可评分的任务。
3. 为每个任务绑定最小证据。
4. 生成“任务 - 能力目标 - 评分维度”对齐表。
5. 将任务完成度拆分到每个任务，不允许笼统给分。
6. 为关系图/关系表生成三档评分规则：
   - 未提交
   - 已提交但关系不足
   - 关系表达完整
7. 将问题分成两类：
   - 安全/合规/诚信红线
   - 维度性扣分
8. 把上一轮高频误区前置转成：
   - 学生版说明单里的提醒
   - 教师验收表里的检查点
   - 评分 rubric 里的重点字段
9. 固定生成 `AI输出审核与人工复核记录（必填）`。
10. 固定生成学生自评表。
11. 固定生成教师验收表。
12. 固定生成诊断标签建议。
13. 固定生成班级级反馈模板。

## Output Contract

默认输出：
1. 项目目标
2. 最小证据
3. 任务说明
4. 任务分值拆分表
5. 关系图/关系表评分规则
6. 红线规则
7. 观察 / 判断 / 证据 / 推理字段
8. 学生版作业说明单
9. AI 输出审核与人工复核记录
10. 学生自评表
11. 教师验收表
12. 评分 Rubric
13. 诊断标签
14. 班级级教学反馈模板

运行落点：
1. 规则与模板层：`40_evaluation/*.md`
2. 章节变量层：`40_evaluation/runtime/chapter_profiles/*.json`
3. 自动生成层：`40_evaluation/runtime/generate_assignment_pack_v5.py`
4. 正式资产层：`50_assets/assignment_packs/`
5. 经验回看层：`10_memory/`

当某章 profile 已稳定时，SK07 的正式输出优先不再停留在抽象模板，而应落到：

- `50_assets/assignment_packs/<chapter_dir>/教师端作业包_v5.0.md`
- `50_assets/assignment_packs/<chapter_dir>/学生版作业说明单_v5.0.md`
- `50_assets/assignment_packs/<chapter_dir>/教师验收表_v5.0.md`
- `50_assets/assignment_packs/<chapter_dir>/班级级教学反馈模板_v5.0.md`

目录总入口见：

- `50_assets/assignment_packs/README_v5.0.md`
- `10_memory/README_v5.0.md`

## Quality Gate

生成后的作业包必须满足：
1. 一目标一证据
2. 一任务一分值
3. 红线问题与一般扣分问题分离
4. 学生能看懂“要做什么、交什么、怎么扣分”
5. 教师能稳定评分
6. 批改后能形成班级级反馈
7. 能看出学生是否审核了 AI 输出，而不是直接复制
8. 能直接落成 `学生版作业说明单模板_v5.0` 与 `教师验收表模板_v5.0`
9. 能看出本轮作业是否已经吸收了上一轮高频误区
