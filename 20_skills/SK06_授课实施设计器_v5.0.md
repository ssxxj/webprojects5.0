# SK06 授课实施设计器 v5.0

## Purpose

把已经稳定的 `5.0 章节讲义`、`chapter profile`、`assignment pack` 和 `10_memory`，
转成教师可以直接上课使用的 `90 分钟课堂实施方案`。

与 `projects4.0` 相比，5.0 的重点不是“把讲义再改写一遍”，而是把课堂设计变成：

`讲义主线 -> 课堂动作 -> 形成性评估 -> 作业收口 -> 观察记录入口`

也就是说，SK06 在 5.0 中承担的是 `教学执行桥`，不是内容生成器。

## Concept

### 1. SK06 在 5.0 中解决什么问题

5.0 现在已经有：

1. `章节讲义`
2. `assignment pack`
3. `评分与反馈`
4. `memory`

但如果没有稳定的课堂实施方案，系统仍会断在：

`讲义很好看 -> 上课还是临场发挥 -> 作业和反馈无法稳定闭环`

所以 SK06 的核心任务是：

把“可读讲义”转成“可教课堂方案”。

### 2. SK06 不是什么

1. 不是再写一份讲义
2. 不是 PPT 提纲拼接器
3. 不是单纯的时间分配表
4. 不是把 OMDM 四个字贴到讲义边上就算完成

## Mechanism

### 1. 输入结构

SK06 默认至少读取这四类上游：

1. `教师版章节讲义`
   提供章节主线、单元结构、概念和误区。
2. `chapter profile`
   提供能力目标、任务分值、专业校验点和红线。
3. `assignment pack`
   提供本章最终作业交付物和收口要求。
4. `10_memory`
   提供上一轮高频误区、标签和案例。

如果四类输入不齐，至少要有：

1. 教师版讲义
2. assignment pack
3. 本章是否已有历史 feedback 的判断

### 2. 核心转换逻辑

SK06 的转换不是“照着讲义顺序讲一遍”，而是做 6 步：

1. 判断本节属于：
   - 基础认知章
   - 信息收集/观察章
   - 漏洞机制章
   - 综合复盘章
2. 从讲义中抽出 `本节唯一主线`
3. 将内容压缩成 `Observation -> Decision -> Evidence -> Mechanism -> Modification`
4. 明确哪些形成性评估与 assignment pack 的最终证据字段一致
5. 明确本节结束后学生应带走什么“最小收口图”
6. 给 Step 9 留出最小课堂观察入口

### 3. 输出结构

SK06 的标准输出不是自由格式，而应固定为：

1. 课堂基础信息
2. 本节唯一主线
3. 课堂目标
4. 先修要求与环境
5. 90 分钟节奏
6. OMDM 与课堂动作对照
7. 板书/图示主骨架
8. 关键提问设计
9. 形成性评估
10. 作业/课后衔接
11. 高频误区与纠偏
12. Step 9 最小观察记录项

### 4. 为什么先做 SK06 再做“授课脚本生成器”

因为生成器只能自动化“写”，不能先定义“该怎么教”。

SK06 先稳定课堂实施规则，后续生成器才能安全复用：

1. 什么叫本节唯一主线
2. 什么叫可执行的 Observation / Decision / Evidence
3. 什么叫和作业证据字段对齐
4. 什么叫课堂结束时必须留下的观察记录

## Use When

在以下情形使用 SK06：

1. 某章 `教师版讲义_v5.0.md` 已稳定
2. 该章 `assignment pack` 已生成
3. 需要产出教师可直接使用的课堂实施方案
4. 需要把“讲义 -> 作业 -> 观察 -> 复盘”真正打通

不适合在以下情形直接使用：

1. 章节讲义还没稳定
2. assignment pack 还没生成
3. 章节目标还在频繁变动

## Input Contract

必须输入：

1. 本章教师版讲义
2. 本章教师端作业包
3. 本章 profile
4. 授课时长
5. 班级基础水平
6. 实验环境

若准备进入后续“逐章授课脚本生成器”，建议改用稳定输入接口：

1. `50_assets/课堂实施方案/逐章授课脚本生成器输入规范_v5.0.md`
2. `50_assets/课堂实施方案/逐章授课脚本输入模板_v5.0.yaml`

建议补充输入：

1. `10_memory/诊断标签索引_v5.0.md`
2. `10_memory/误区库_v5.0.md`
3. `10_memory/案例库_v5.0.md`
4. `10_memory/班级教学反馈索引_v5.0.md`

### 稳定输入接口说明

SK06 的手工模式仍可直接读取讲义、assignment pack、profile 和 memory。

但如果后续要做“逐章授课脚本生成器”，不应直接自由读取整份讲义全文，而应先把输入压成 6 层稳定字段：

1. 系统元数据层
2. 章节主线层
3. 班级与环境层
4. 课堂动作层
5. 作业对齐层
6. 观察与复盘层

稳定接口的目的，是先固定“怎么教”，再自动化“怎么写”。

## Process

1. 先判定本章类型：基础章 / 观察章 / 漏洞章 / 综合章。
2. 从教师版讲义中抽取本节唯一主线，不得超过 1 句。
3. 从讲义中抽出本节最关键的：
   - 现象
   - 强制判断题
   - 证据动作
   - 机制收束
   - 条件变化题
4. 将以上内容压成 90 分钟可执行节奏：
   - 10 分钟开场与边界
   - 20 分钟 Concept
   - 25 分钟 Mechanism
   - 25 分钟 Application
   - 10 分钟形成性评估与收口
5. 从 assignment pack 中回拉：
   - 本节最后应服务哪一个任务
   - 哪个关系图/关系表要前置
   - 哪个 AI 审核提醒要前置
6. 从 memory 中前置：
   - 上轮最常见误区
   - 哪个结论最容易超范围
   - 哪类证据学生最容易漏
7. 生成标准课堂实施方案。
8. 明确 Step 9 观察记录至少保留哪 3 项。
9. 如果输入已经压成 YAML/JSON 结构化字段，可直接调用：
   - `50_assets/课堂实施方案/runtime/generate_lesson_script_v5.py`

## Output Contract

默认输出应直接落成一份 `课堂实施方案_v5.0.md`，包含：

1. 课堂基础信息
2. 本节唯一主线
3. 课堂目标
4. 先修要求与环境
5. 90 分钟节奏表
6. OMDM 与课堂动作对照表
7. 板书/图示主骨架
8. 关键提问与强制选择题
9. 形成性评估点
10. 作业/课后衔接
11. 高频误区与纠偏动作
12. Step 9 最小观察记录

推荐落点：

1. 正式章节资产层：
   - `50_assets/课堂实施方案/<chapter_dir>/`
2. 若先做草稿：
   - `50_assets/课堂实施方案模板_v5.0.md`
3. 若进入自动化生成：
   - `50_assets/课堂实施方案/runtime/generate_lesson_script_v5.py`

## Recommended Output Shape

1. 课堂基础信息
2. 本节唯一主线
3. 课堂目标
4. 先修要求
5. 环境与边界
6. 时间分配
7. OMDM 对照
8. 课堂动作脚本
9. 板书与图示设计
10. 形成性评估
11. 作业收口
12. 高频误区与纠偏
13. 最小观察记录

## Quality Gate

合格的 SK06 输出至少满足：

1. `90 分钟内可执行`
2. `只有 1 条本节唯一主线`
3. 至少有 `1 个先判断再验证` 的课堂动作
4. 至少有 `1 个要求学生指出证据字段` 的中间动作
5. 至少有 `1 个形成性评估点`
6. 至少有 `1 个条件变化问题`
7. 至少有 `1 个和 assignment pack 直接对齐的收口动作`
8. 至少有 `3 个 Step 9 观察项`
9. 不得把课堂设计成“只演示操作、不解释机制”

## Failure Handling

1. 如果内容过多：
   优先保留主线、强制判断题、证据动作、形成性评估，压缩拓展案例。
2. 如果学生基础过弱：
   Decision 环节先收缩成 2 个候选解释，不要一开始给太多分支。
3. 如果学生只会回答结论：
   先加“指出哪个字段支持你”的中间动作，不继续追加定义讲解。
4. 如果本章暂无真实 feedback：
   允许先使用课程级 memory 高频误区，等第一轮正式授课后再回归修订。
