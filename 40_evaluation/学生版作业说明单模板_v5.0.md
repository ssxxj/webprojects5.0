# 学生版作业说明单模板 v5.0

## 1. 定位

本模板用于把第 11 步生成的教师端作业包，压缩为学生可直接执行的提交说明。

`v5.0` 的变化不是多加一张说明页，而是把学生端要求改成：

1. 你要学什么
2. 你要完成什么
3. 你要交什么
4. 你为什么这样判断
5. 你是否审核过 AI 输出
6. 哪些问题只是扣分，哪些问题会触发红线

## 2. 生成原则

- 面向学生语言，不写教师内部裁量语气
- 默认按 `观察 -> 判断 -> 证据 -> 推理` 组织
- 默认说明 `任务分值`，避免学生不知道缺一项会丢多少
- 默认说明 `20 分任务完成度子分 + 100 分课程总评` 的关系，避免学生把任务表误读成最终总分
- 默认保留 `关系图/关系表` 要求
- 默认保留 `AI输出审核与人工复核记录（必填）`
- 默认前置 `关键字段示例` 与 `证据 -> 结论` 说明
- 红线问题只写安全、合规、诚信边界
- 自评表、关键字段、局部字段不全属于扣分项，不写成一刀切退回
- 若 `10_memory` 已有相近章节高频误区，应把误区提醒写进学生版说明单，而不是只留给教师端

## 4. 章节变量替换规则

### 4.1 目标变量

把本章目标写成学生能理解的能力句，而不是教师内部术语。

推荐写法：

- 完成本次作业后，你应能区分……
- 完成本次作业后，你应能解释……
- 完成本次作业后，你应能根据证据判断……

### 4.2 任务变量

每个任务至少写清 3 件事：

1. 目标是什么
2. 最小字段是什么
3. 这个任务值多少分

如果本章高频问题集中在 `E1 / E2 / T1 / T2`，建议额外写出：

4. 关键字段示例是什么
5. 什么样的结论超出了证据范围

另外必须在学生端明确写出：

6. 本章任务表合计的 20 分只对应“任务完成度”子分
7. 本章最终成绩按 100 分制总评计算

### 4.3 关系图/关系表变量

不同章节可以是不同形态，例如：

- 信息层 -> 方法 -> 结果类型 -> 下一步
- 输入 -> 处理 -> 输出 -> 风险
- 现象 -> 机制 -> 控制点 -> 防护

但必须写清“关系”，不能退化成单纯工具介绍。

推荐显式提示学生：

- 错误写法：只列工具、术语、概念
- 合格写法：至少写出“上一层结果为什么导向下一步”
- 更好写法：写出“能说明什么 / 不能替代什么 / 下一步为什么是这个”

### 4.4 AI审核变量

本模块不是问学生“有没有用 AI”，而是看学生是否：

1. 审核过 AI 输出
2. 删改过 AI 输出
3. 独立核实过专业判断
4. 能指出 AI 最容易出错的边界

生成时建议额外提醒学生：

- 不能只写“已审核”“已检查”，必须写清保留、删改和核实内容
- 若未使用 AI，也要写清自己主动放弃了哪些思路，以及为什么放弃

### 4.5 高频误区前置提醒

如果 `10_memory` 已经显示本章或相近章节有高频误区，建议在学生说明单中直接前置 3 到 4 条提醒，例如：

1. 关系图/关系表不是工具清单，必须体现关系链。
2. 截图不是越多越好，关键字段必须能看清。
3. 线索不等于证据，端口不等于漏洞，第三方索引不等于实时探测。
4. AI 可以用，但不审核就等于没有完成本项要求。

## 5. 质量门

生成后的学生版作业说明单必须满足：

1. 学生一眼能看出“要学什么”和“要交什么”
2. 学生知道缺一个任务会丢多少分
3. 学生知道关系图/关系表不是可有可无
4. 学生知道红线问题与一般扣分问题的区别
5. 学生知道 AI 可以用，但必须审核，不能直接复制
6. 学生知道“20 分任务表”和“100 分总评”的关系
7. 学生至少能看懂其他六个维度的大致评分点
8. 可直接发给学生使用
9. 能提前规避上一轮已经出现过的高频误区
