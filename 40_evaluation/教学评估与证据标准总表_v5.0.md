# 教学评估与证据标准总表 v5.0

## 1. 最小证据集合

所有作业默认要求：

1. 证据与任务逐项对应
2. 关键截图可定位字段
3. 结果能支撑判断，而不只是证明“做过”
4. 必须有机制解释
5. 必须有至少一个下一步或条件变化推理
6. 必须有学生自评
7. 必须有 `AI输出审核与人工复核记录（必填）`

## 2. 红线规则

以下问题允许直接压到 60 分以下：

1. 未脱敏
2. 超出授权边界
3. 伪造、代做、抄袭
4. 与他人作业高度雷同
5. 课程明确禁止的违规操作

注：
自动检测可以触发高风险标记，但纪律性处理应由教师复核确认。

## 3. 非红线问题的处理规则

以下问题不得直接封顶，必须按维度扣分：

1. 任务缺失或只完成部分
2. 关键字段不全
3. 综合关系图/表关系表达不足
4. 自评表缺失
5. 机制解释较弱
6. 证据一般但并非不存在

## 4. 统一质量评分底座

默认使用七维：

| 维度 | 默认权重 | 核心判断 |
| --- | --- | --- |
| 任务完成度 | 20% | 任务是否真实完成 |
| 证据质量 | 18% | 证据是否清楚、可定位 |
| 机制解释 | 20% | 是否讲清因果链 |
| 关系链与下一步 | 12% | 是否能从当前结果推到合理下一步 |
| 风险与防护 | 10% | 是否能说明风险和控制 |
| 表达与结构 | 10% | 是否清楚、可读、可验收 |
| 专业校验与责任意识 | 10% | 是否区分层次、避免绝对化、边界表达稳妥 |

## 5. 关系图/关系表口径

统一按三档：
1. 未提交：0
2. 已提交但关系不足：部分分
3. 关系表达完整：足额分

## 6. AI 输出审核与人工复核记录

默认作为必交项。

至少包含：
1. 是否使用 AI
2. AI 帮助的环节
3. 保留了哪 2 处 AI 输出，为什么
4. 删改了哪 2 处 AI 输出，为什么
5. 自行核实过的 3 个专业判断
6. 最容易误判的一处边界问题

## 7. 输出层要求

### 学生端
输出：
1. 等级
2. 做得好的地方
3. 需要完善的地方
4. 如何完善

### 教师端
输出：
1. 分项分
2. 扣分依据
3. 诊断标签
4. 高风险标记

### 班级端
输出：
1. 班级平均分
2. 各维度平均分
3. 高频错误标签
4. 下一轮教学建议
