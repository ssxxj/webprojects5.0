# 10_memory v5.0 说明

本目录用于沉淀 5.0 运行后的长期资产。

## 当前定位

`10_memory` 不存放“临时过程文件”，只存放值得复用的诊断与教学经验。

这里优先沉淀：
1. 诊断标签索引
2. 高频误区库
3. 正反样本与案例说明
4. 班级级教学反馈索引
5. 后续可迁移到讲义、作业包、profile 或脚本的升级线索

## 当前文件

1. [诊断标签索引_v5.0.md](诊断标签索引_v5.0.md)
   统一说明各标签代表什么、如何识别、下一步应补讲还是应改系统。
2. [误区库_v5.0.md](误区库_v5.0.md)
   沉淀高频误区、典型症状、教学动作和系统升级建议。
3. [案例库_v5.0.md](案例库_v5.0.md)
   记录可作为正例、反例或示范讲评的案例类型。
4. [班级教学反馈索引_v5.0.md](班级教学反馈索引_v5.0.md)
   汇总各班各章的反馈输出、共性问题和可迁移结论。
5. [runtime/memory_update_v5.py](runtime/memory_update_v5.py)
   根据一个或多个 `教学反馈摘要_v5.json` 自动重建记忆层索引与沉淀文件。
6. [memory_snapshot_v5.json](memory_snapshot_v5.json)
   自动生成的记忆层摘要快照。

## 首轮回填状态

目前已根据第二章两班的正式 v5 教学反馈完成首轮回填：

1. `24信安01 第二章`
2. `24信安02 第二章`

这意味着 `10_memory` 现在不再是空壳，已经可以支撑：

1. 下次课补讲点提取
2. 作业模板修订

## 自动更新入口

当已经生成一个或多个 `教学反馈摘要_v5.json` 后，可直接运行：

```bash
python3 10_memory/runtime/memory_update_v5.py \
  --inputs <摘要1.json> <摘要2.json>
```

默认会更新：

1. `班级教学反馈索引_v5.0.md`
2. `误区库_v5.0.md`
3. `案例库_v5.0.md`
4. `memory_snapshot_v5.json`
