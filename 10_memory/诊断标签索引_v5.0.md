# 诊断标签索引 v5.0

## 1. 用途

诊断标签用于把“长评语”压缩成可统计、可迁移、可回写系统的结构化判断。

原则：
1. 标签用于诊断，不直接替代纪律裁决。
2. 同一作业可命中多个标签。
3. 标签既服务评分，也服务课后复盘和模板升级。

## 2. 当前标签

| 标签 | 含义 | 典型触发 | 首选动作 |
| --- | --- | --- | --- |
| `T1` | 任务字段缺失 | 任务已写但关键字段明显缺失 | 先补任务说明与字段示例 |
| `T2` | 任务部分完成 | 任务存在，但完成度不足 | 先补任务链条与最小证据 |
| `E1` | 证据不清晰 | 截图、标注、字段定位不足 | 先补“关键字段示例” |
| `E2` | 证据与结论不对应 | 结论超过证据支持范围 | 补讲“证据 -> 结论”映射 |
| `M1` | 机制解释不足 | 会操作，但说不清为什么 | 补讲机制模板 |
| `R1` | 关系图/关系表缺失 | 收口项未提交 | 强化提交清单与验收表 |
| `R2` | 关系链不足 | 提交了图/表，但没体现链条 | 示范“现象 -> 机制 -> 控制 -> 下一步” |
| `B1` | 授权边界表达不足 | 没明确授权、脱敏、禁止动作 | 前置强调边界与合规 |
| `P1` | 防护建议泛化 | 建议停留在口号层 | 补讲“根因层/暴露面/后果范围” |
| `C1` | 高度雷同待复核 | 与同班作业结构和文本高度重复 | 教师人工复核后再判定 |

## 3. 标签判读原则

1. `T1/T2` 优先说明任务层问题，不直接等于低专业度。
2. `E1/E2` 优先说明证据链问题，常与 `R2` 联动出现。
3. `M1` 说明学生会做但未形成稳定解释框架。
4. `R1/R2` 是 Step 11 收口项质量的直接信号。
5. `B1` 和 `C1` 具有更高管理优先级，但 `C1` 仍需教师确认。
