# 案例库 v5.0

## 1. 用途

案例库服务两件事：

1. 课堂示范讲评
2. 模板与规则升级时的证据参考

## 2. 当前记录方式

本阶段先不复制学生全文，而是记录“案例类型 + 来源班级 + 可借鉴点”，避免记忆层变成原始作业堆积区。

## 3. 自动沉淀的正例样本

### C-001 第二章 Web信息收集与请求分析 高质量完成样本

- 来源：
  - `24信安01`：蒋子静、秦媛媛、刘俊妤
  - `24信安02`：梁淑岚、唐雨欣、陆海婷
- 共同特征：
  - 主体任务完成较稳定
  - 证据或关键字段定位相对清楚
  - 机制解释相对完整
- 适合用途：
  - 作为结构型正例
  - 作为教师讲评时的完成度参考

### C-002 高度雷同待复核类样本

- 来源：
  - `第二章 Web信息收集与请求分析` / `24信安02`：4 例
- 适合用途：
  - 用于讲诚信边界
  - 用于提醒“自动检测只做高风险提示，最终仍需教师确认”

### C-003 补交管理类样本

- 来源：
  - `第二章 Web信息收集与请求分析` / `24信安01`：未提交 8 人
  - `第二章 Web信息收集与请求分析` / `24信安02`：未提交 2 人
- 适合用途：
  - 用于改进截止前核对表与最小提交清单

## 4. 当前边界

若要把案例库进一步升级成真正的讲评库，后续还应补入：
1. 对应章节
2. 对应标签
3. 对应推荐讲法
4. 对应可回写的模板位置
