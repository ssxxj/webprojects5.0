# 误区库 v5.0

## 1. 记录原则

误区库只记录“会反复出现、值得回写系统”的问题，不记录一次性个案。

每条误区至少写清：
1. 误区是什么
2. 典型症状
3. 最可能来源
4. 下轮教学动作
5. 需要升级的系统文件

## 2. 自动沉淀结果

### M-001 关系链不足

- 对应标签：`R2`
- 观察来源：
  - `第二章 Web信息收集与请求分析` / `24信安01`：12 人次
  - `第二章 Web信息收集与请求分析` / `24信安02`：17 人次
- 典型症状：
  - 图/表已提交，但只列工具或概念，没有写“能说明什么 / 不能替代什么 / 下一步”。
  - 只写结果列表，没有过程或因果链。
- 最可能来源：
  - 教师讲评中缺少完整关系链示范。
  - 学生不清楚关系图与工具清单的区别。
- 下轮教学动作：
  - 增加一次关系链示范讲评，演示“现象 -> 机制 -> 控制点 -> 下一步”。
- 需升级文件：
  - `40_evaluation/学生版作业说明单模板_v5.0.md`
  - `40_evaluation/教师验收表模板_v5.0.md`

### M-002 证据不清晰

- 对应标签：`E1`
- 观察来源：

  - `第二章 Web信息收集与请求分析` / `24信安02`：11 人次
- 典型症状：
  - 截图很多，但没有标出关键字段。

  - 模板没有把关键字段示例写在足够靠前的位置。
- 下轮教学动作：
  - 补讲“证据最小集”，明确各任务至少应标出哪些字段。
- 需升级文件：
  - `40_evaluation/第11步作业设计模板_v5.0.md`

### M-003 任务部分完成

- 对应标签：`T2`
- 观察来源：
  - `第二章 Web信息收集与请求分析` / `24信安01`：6 人次
  - `第二章 Web信息收集与请求分析` / `24信安02`：11 人次
- 典型症状：
  - 主体任务已写，但仍缺少一个或多个关键判断点。
  - 能做出结果，但不能把结果解释清楚。
- 最可能来源：
  - 学生只完成了操作层，没有完成解释层。
  - 任务说明对“完成”和“达标”区分不够清楚。
- 下轮教学动作：
  - 在下轮讲评中增加“已完成但未达标”的反例示范。
- 需升级文件：
  - `40_evaluation/教师验收表模板_v5.0.md`

### M-005 证据与结论不对应

- 对应标签：`E2`
- 观察来源：
  - `第二章 Web信息收集与请求分析` / `24信安01`：5 人次
  - `第二章 Web信息收集与请求分析` / `24信安02`：9 人次
- 典型症状：
  - 把线索写成证据，把端口写成漏洞，把第三方索引写成实时探测。
  - 结论明显超出截图或抓包内容本身能支持的范围。
- 最可能来源：
  - 线索、证据、主动探测三层没有真正分清。
- 下轮教学动作：
  - 补讲“线索 / 证据 / 主动探测”的分层关系和边界。
- 需升级文件：
  - `40_evaluation/第11步作业设计模板_v5.0.md`
  - `40_evaluation/runtime/chapter_profiles/`

### M-006 防护建议泛化

- 对应标签：`P1`
- 观察来源：
  - `第二章 Web信息收集与请求分析` / `24信安01`：10 人次
  - `第二章 Web信息收集与请求分析` / `24信安02`：3 人次
- 典型症状：
  - 建议停留在“加强管理、提高意识”层面。
  - 没有区分根因控制、暴露面控制和后果范围控制。

- 下轮教学动作：

- 需升级文件：

### M-007 任务字段缺失

## 3. 当前结论

当前更值得优先修的，不是“增加更多工具”，而是：
1. 关系链表达
2. 证据到结论映射
3. AI 输出审核与人工复核记录
